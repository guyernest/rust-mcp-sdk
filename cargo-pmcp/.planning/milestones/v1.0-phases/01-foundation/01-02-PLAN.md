**GREEN phase — Implement McpClient:**

```rust
use reqwest::Client;
use serde_json::{json, Value};
use std::time::Duration;
use crate::loadtest::error::McpError;

/// MCP-aware HTTP client for load testing.
/// Each virtual user owns one instance with its own session.
pub struct McpClient {
    http: Client,
    base_url: String,
    session_id: Option<String>,
    request_timeout: Duration,
    next_request_id: u64,
}
```

Implement:
- `McpClient::new(http: Client, base_url: String, timeout: Duration) -> Self` — Takes a reqwest::Client by value (allows shared client from Phase 2 via Clone). Starts with no session and next_request_id=1.
- `fn next_id(&mut self) -> u64` — Returns current id, increments.
- `fn session_id(&self) -> Option<&str>` — Returns session_id as ref.
- `fn build_initialize_body(&mut self) -> Value` — Constructs the initialize JSON-RPC request using serde_json::json!. Protocol version "2025-06-18". Client name "cargo-pmcp-loadtest", version from env!("CARGO_PKG_VERSION").
- `fn build_initialized_notification(&self) -> Value` — JSON-RPC notification (no id field). Method "notifications/initialized".
- `fn build_tool_call_body(&mut self, tool: &str, arguments: &Value) -> Value` — Method "tools/call", params.name, params.arguments.
- `fn build_resource_read_body(&mut self, uri: &str) -> Value` — Method "resources/read", params.uri.
- `fn build_prompt_get_body(&mut self, prompt: &str, arguments: &std::collections::HashMap<String, String>) -> Value` — Method "prompts/get", params.name, params.arguments.
- `fn extract_session_id(&mut self, headers: &reqwest::header::HeaderMap)` — Reads "mcp-session-id" header, stores in self.session_id.
- `fn parse_response(body: &[u8]) -> Result<Value, McpError>` — Parses JSON, checks for error object, returns result field or McpError::JsonRpc.

Async methods (send over HTTP):
- `pub async fn initialize(&mut self) -> Result<Value, McpError>` — Builds init body, POSTs to base_url with headers (Content-Type: application/json, Accept: application/json, text/event-stream), extracts session_id from response headers, parses JSON-RPC response, then sends initialized notification with session_id attached. Returns the initialize result.
- `pub async fn call_tool(&mut self, tool: &str, arguments: &Value) -> Result<Value, McpError>` — Builds body, POSTs with session_id header, parses response.
- `pub async fn read_resource(&mut self, uri: &str) -> Result<Value, McpError>` — Same pattern for resources/read.
- `pub async fn get_prompt(&mut self, prompt: &str, arguments: &std::collections::HashMap<String, String>) -> Result<Value, McpError>` — Same pattern for prompts/get.
- Private `async fn send_request(&mut self, body: &Value) -> Result<(reqwest::header::HeaderMap, bytes::Bytes), McpError>` — Shared HTTP send logic. Attaches session_id if present. Applies per-request timeout. Classifies reqwest errors via McpError::classify_reqwest. Checks HTTP status for non-2xx -> McpError::Http. **CRITICAL latency timing boundary:** Capture `end_time = Instant::now()` after `response.bytes().await` completes but BEFORE `serde_json::from_slice()` or any JSON parsing. The caller uses `start_time` and `end_time` to compute latency. JSON parse time must NOT be included in the latency measurement.

**CRITICAL per user decision:** Per-request timeout MUST use `reqwest::RequestBuilder::timeout()` (not client-level default only). Session failure handling (retry once then mark dead) is Phase 2 scope -- this plan just does the basic request cycle.
