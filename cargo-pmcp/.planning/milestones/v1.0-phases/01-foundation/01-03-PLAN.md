<objective>
Create an HdrHistogram-based metrics pipeline with coordinated omission correction, separate success/error buckets, and per-operation-type tracking.

Purpose: Accurate latency measurement is the core value proposition of a load tester. This plan bakes in coordinated omission correction from day one (not retrofittable per research) and separates success from error latency for clean signal. The MetricsRecorder API is designed for single-owner usage, ready for Phase 2's mpsc channel aggregation pattern.

Output: `src/loadtest/metrics.rs` with MetricsRecorder, RequestSample, OperationType, MetricsSnapshot types, and comprehensive tests proving percentile accuracy and coordinated omission correction work.
</objective>

Key design decisions from CONTEXT.md:
- Every MCP request counts toward throughput (initialize, tools/call, resources/read, prompts/get)
- Success latency and error latency tracked in separate buckets
- Time resolution: milliseconds
- Coordinated omission correction via record_correct() at recording time
- The weighted mix and summary report break down metrics per MCP operation type

From RESEARCH.md anti-patterns:
- NEVER use Arc<Mutex<Histogram>> — single-owner design, Phase 2 uses mpsc channels
- NEVER use Histogram::new_with_max() — use Histogram::new(sigfig) for auto-resizing
- NEVER include JSON parse time in latency measurement (timing concern for client, not metrics)

5. `test_percentiles_known_distribution` — Record 100 success samples with durations 1ms through 100ms (i.e., sample i has duration i ms). Assert:

**GREEN phase — Implement MetricsRecorder:**

```rust
use hdrhistogram::Histogram;
use std::collections::HashMap;
use std::time::{Duration, Instant};
use crate::loadtest::error::McpError;
```

Implement `OperationType`:
- Derive `Debug, Clone, Copy, PartialEq, Eq, Hash`
- Implement `std::fmt::Display` — Initialize->"initialize", ToolsCall->"tools/call", ResourcesRead->"resources/read", PromptsGet->"prompts/get", ToolsList->"tools/list", ResourcesList->"resources/list", PromptsList->"prompts/list"

Implement `RequestSample`:
- Fields: operation (OperationType), duration (Duration), result (Result<(), McpError>), timestamp (Instant)
- Constructor: `RequestSample::success(op, duration)` and `RequestSample::error(op, duration, error)` for convenience

Implement `MetricsRecorder`:
- Fields:
  - `success_histogram: Histogram<u64>` — created with `Histogram::new(3)` (3 significant figures, auto-resizing)
  - `error_histogram: Histogram<u64>` — same
  - `expected_interval_ms: u64`
  - `operation_counts: HashMap<OperationType, u64>` — per-operation request counts
  - `error_counts: HashMap<OperationType, u64>` — per-operation error counts
  - `total_success: u64`
  - `total_errors: u64`

- `MetricsRecorder::new(expected_interval_ms: u64) -> Self`
- `fn record(&mut self, sample: &RequestSample)`:
  - Increment operation_counts for sample.operation
  - Get duration as `sample.duration.as_millis() as u64`
  - If result is Ok: `self.success_histogram.record_correct(ms, self.expected_interval_ms)`, increment total_success
  - If result is Err: `self.error_histogram.record_correct(ms, self.expected_interval_ms)`, increment total_errors, increment error_counts
  - Use `.ok()` on record_correct (ignore overflow — auto-resizing handles it)

- Percentile methods (on success histogram): `p50()`, `p95()`, `p99()` — return u64 (milliseconds). Call `self.success_histogram.value_at_quantile(0.50/0.95/0.99)`. Return 0 if histogram is empty (check `len() == 0`).
- Error percentile methods: `error_p50()`, `error_p95()`, `error_p99()` — same for error histogram.
- Count methods: `success_count() -> u64`, `error_count() -> u64`, `total_requests() -> u64`
- `error_rate() -> f64` — total_errors as f64 / total_requests as f64. Return 0.0 if total is 0.
- `operation_count(op: OperationType) -> u64` — lookup in operation_counts, default 0.

Implement `MetricsSnapshot`:
- Fields: p50, p95, p99, error_p50, error_p95, error_p99, success_count, error_count, total_requests, error_rate, operation_counts (HashMap), per_operation_errors (HashMap)
- `MetricsRecorder::snapshot(&self) -> MetricsSnapshot` — captures current state
- Derive `Debug, Clone` on MetricsSnapshot

Run tests — they MUST all pass. Commit.
