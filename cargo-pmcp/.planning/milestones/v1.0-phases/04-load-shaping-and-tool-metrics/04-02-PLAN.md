<objective>
Extend the metrics pipeline with per-tool-name HdrHistogram tracking so that latency, throughput, and error metrics are reported individually for each MCP tool, resource, and prompt in both terminal summary and JSON report.

Purpose: Overall aggregate metrics hide which specific MCP tools are slow or failing. Developers need per-tool breakdowns to identify which operations are bottlenecks or error-prone under load, enabling targeted optimization of their MCP server.

Output: `ToolMetrics` and `ToolSnapshot` types in metrics.rs, tool_name extraction in vu.rs, per-tool table section in summary.rs, per-tool extended metrics in JSON report with schema version 1.1.
</objective>
