<objective>
Implement a rolling-window breaking point detector that auto-detects when an MCP server's performance degrades under load, surfacing the detection as a live terminal warning and a structured field in the JSON report.

Purpose: Without automatic degradation detection, developers must manually inspect metrics to find the VU count where their server begins to fail. The breaking point detector automates this analysis, giving developers a clear signal: "Your server began degrading at N VUs because of X." This is the highest-value feature in Phase 4 for developers iterating on server performance.

Output: `breaking.rs` module with `BreakingPointDetector`, engine integration, live terminal warning, JSON report field, property tests, and fuzz target.
</objective>

2. Add `pub mod breaking;` to `src/loadtest/mod.rs`.

3. Define core types:

```rust
use std::collections::VecDeque;
use std::time::Instant;

/// Default rolling window size (number of snapshot samples).
const DEFAULT_WINDOW_SIZE: usize = 10;

/// Minimum error rate threshold before detection triggers.
const ERROR_RATE_ABSOLUTE_THRESHOLD: f64 = 0.10; // 10%

/// Error rate must be this multiple of baseline to trigger.
const ERROR_RATE_RELATIVE_MULTIPLIER: f64 = 2.0;

/// P99 latency must be this multiple of baseline to trigger.
const P99_RELATIVE_MULTIPLIER: f64 = 3.0;

/// A single snapshot sample stored in the rolling window.
#[derive(Debug, Clone)]
struct WindowSample {
    error_rate: f64,
    p99_ms: u64,
    active_vus: u32,
    timestamp: Instant,
}

/// A detected breaking point event.
#[derive(Debug, Clone)]
pub struct BreakingPoint {
    /// VU count at which the breaking point was detected.
    pub vus: u32,
    /// Reason category: "error_rate_spike" or "latency_degradation".
    pub reason: String,
    /// Human-readable detail string.
    pub detail: String,
    /// When the breaking point was detected.
    pub detected_at: Instant,
}
```

4. Implement `BreakingPointDetector`:

```rust
pub struct BreakingPointDetector {
    window: VecDeque<WindowSample>,
    window_size: usize,
    detected: bool,
    breaking_point: Option<BreakingPoint>,
}
```

5. Implement methods:

- `pub fn new(window_size: usize) -> Self` -- creates detector with given window size and empty state.
- `pub fn with_default_window() -> Self` -- creates with `DEFAULT_WINDOW_SIZE`.
- `pub fn observe(&mut self, snapshot: &MetricsSnapshot, active_vus: u32) -> Option<BreakingPoint>`:
  a. Create `WindowSample` from snapshot (`error_rate`, `p99`, `active_vus`, `Instant::now()`).
  b. Push to back of `window`. If window exceeds `window_size`, pop front.
  c. If already `detected`, return `None` (fire once only).
  d. If window is not full (`window.len() < window_size`), return `None` (need minimum fill).
  e. Split window in half: `baseline = window[0..half]`, `recent = window[half..]`.
  f. Compute baseline averages: `baseline_error_rate = mean(baseline.error_rate)`, `baseline_p99 = mean(baseline.p99_ms)`.
  g. Compute recent averages: `recent_error_rate = mean(recent.error_rate)`, `recent_p99 = mean(recent.p99_ms)`.
  h. **Error rate check:** If `recent_error_rate > ERROR_RATE_ABSOLUTE_THRESHOLD` AND `recent_error_rate > baseline_error_rate * ERROR_RATE_RELATIVE_MULTIPLIER`, detect with reason `"error_rate_spike"`. Detail: `format!("Error rate {:.1}% exceeds threshold (>10% and >{}x baseline {:.1}%)", recent_error_rate * 100.0, ERROR_RATE_RELATIVE_MULTIPLIER, baseline_error_rate * 100.0)`.
  i. **Latency check:** If `baseline_p99 > 0` (avoid div-by-zero) AND `recent_p99 > baseline_p99 * P99_RELATIVE_MULTIPLIER as u64`, detect with reason `"latency_degradation"`. Detail: `format!("P99 {}ms exceeds {}x baseline {}ms", recent_p99, P99_RELATIVE_MULTIPLIER, baseline_p99)`.
  j. If detected, set `self.detected = true`, store `breaking_point`, return `Some(bp)`.

- `pub fn detected(&self) -> bool` -- returns whether a breaking point has been detected.
- `pub fn breaking_point(&self) -> Option<&BreakingPoint>` -- returns the detected breaking point if any.

6. Add comprehensive unit tests:

- `test_no_detection_below_threshold` -- feed 10 samples with low error rate, verify no detection.
- `test_detection_on_error_rate_spike` -- feed 5 baseline samples with 2% error rate, then 5 with 15% error rate, verify detection with correct reason and VU count.
- `test_detection_on_latency_spike` -- feed 5 baseline samples with P99=50ms, then 5 with P99=200ms (>3x), verify detection.
- `test_no_detection_before_window_full` -- feed fewer than window_size samples even with bad metrics, verify no detection.
- `test_fires_only_once` -- trigger detection, then feed more bad data, verify observe returns None after first detection.
- `test_no_detection_when_error_rate_below_absolute` -- 5% error rate even if 3x baseline still should not trigger (below 10% absolute).
- `test_no_detection_when_error_rate_below_relative` -- 11% error rate but baseline was 8% (not >2x), should not trigger.
- `test_detection_returns_correct_vus` -- verify the active_vus from the most recent sample when detection triggers.
- `test_detected_and_breaking_point_accessors` -- verify detected() and breaking_point() return correct state.
