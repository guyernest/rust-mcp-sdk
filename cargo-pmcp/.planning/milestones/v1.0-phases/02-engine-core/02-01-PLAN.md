**Step 3: Create src/loadtest/vu.rs**

This file implements the VU task loop. Each VU:
1. Creates its own McpClient and performs initialize handshake
2. Performs per-VU discovery (tools/list, resources/list, prompts/list calls)
3. Loops executing weighted-random scenario steps until cancellation or iteration limit
4. On session failure mid-test: respawn with exponential backoff (base 500ms, x2 per attempt, +/-25% jitter), max 3 attempts before permanent death
5. Sends RequestSample values through a bounded mpsc channel

**Active VU counter** -- a lightweight atomic wrapper for tracking live VU count:
```rust
#[derive(Clone)]
pub struct ActiveVuCounter(Arc<AtomicU32>);

impl ActiveVuCounter {
    pub fn new() -> Self { Self(Arc::new(AtomicU32::new(0))) }
    pub fn increment(&self) { self.0.fetch_add(1, Ordering::Relaxed); }
    pub fn decrement(&self) { self.0.fetch_sub(1, Ordering::Relaxed); }
    pub fn get(&self) -> u32 { self.0.load(Ordering::Relaxed) }
}
```

**Main VU loop function:**
```rust
pub async fn vu_loop(
    vu_id: u32,
    config: Arc<LoadTestConfig>,
    http_client: Client,        // Shared connection pool via Clone
    base_url: String,
    sample_tx: mpsc::Sender<RequestSample>,
    cancel: CancellationToken,
    iteration_counter: Option<Arc<AtomicU64>>,
    max_iterations: Option<u64>,
    active_vus: ActiveVuCounter,
) { ... }
```

The function body:
1. Call `active_vus.increment()` at entry.
2. Create `McpClient::new(http_client, base_url, config.settings.timeout_as_duration())`.
3. **Initialize phase**: Call `client.initialize()`. Time it and send a RequestSample(OperationType::Initialize) through sample_tx. If initialize fails, attempt respawn with backoff. If all respawn attempts fail, decrement active_vus and return (permanent death).
4. **Discovery phase**: No separate discovery calls needed. The initialize handshake already returns server capabilities. Proceed directly to the load generation loop.
5. **Load generation loop**:
   - Build `WeightedIndex` from `config.scenario.iter().map(|s| s.weight())`. If construction fails (e.g., all zero weights), log error and return.
   - Create `let mut rng = rand::rng();`
   - Loop:
     a. Check `cancel.is_cancelled()` -- break if true. (Quick pre-flight check.)
     b. If `max_iterations` is Some, do `fetch_add(1, Relaxed)` on iteration_counter. If returned value >= max, call `cancel.cancel()` and break. Minor overshoot by up to N-1 VUs is acceptable.
     c. Select step via `dist.sample(&mut rng)`.
     d. Capture `let start = Instant::now();`.
     e. Execute the step via `execute_step(&mut client, &config.scenario[step_idx]).await`.
     f. Capture `let duration = start.elapsed();`.
     g. Build the RequestSample based on result.
     h. Send via `sample_tx.send(sample).await`. If send fails (receiver dropped), break.
     i. On McpError during step execution: record the error sample, then check if the error is a session-fatal error (Connection or Timeout). If so, attempt respawn (create new McpClient, re-initialize). If respawn fails after MAX_RESPAWN_ATTEMPTS, decrement active_vus and return.
6. Decrement `active_vus` at exit (use a drop guard or explicit decrement in all return paths).

**execute_step function:**
```rust
async fn execute_step(client: &mut McpClient, step: &ScenarioStep) -> (OperationType, Result<(), McpError>) {
    match step {
        ScenarioStep::ToolCall { tool, arguments, .. } => {
            let result = client.call_tool(tool, arguments).await;
            (OperationType::ToolsCall, result.map(|_| ()))
        }
        ScenarioStep::ResourceRead { uri, .. } => {
            let result = client.read_resource(uri).await;
            (OperationType::ResourcesRead, result.map(|_| ()))
        }
        ScenarioStep::PromptGet { prompt, arguments, .. } => {
            let result = client.get_prompt(prompt, arguments).await;
            (OperationType::PromptsGet, result.map(|_| ()))
        }
    }
}
```

**respawn_with_backoff function:**
```rust
async fn respawn_with_backoff(attempt: u32, cancel: &CancellationToken) -> bool {
    if attempt >= MAX_RESPAWN_ATTEMPTS {
        return false;
    }
    let backoff = Duration::from_millis(BASE_BACKOFF_MS * 2u64.pow(attempt));
    let jitter_range = backoff.as_millis() as u64 / 4;
    let jitter = if jitter_range > 0 {
        rand::rng().random_range(0..=jitter_range * 2) as i64 - jitter_range as i64
    } else {
        0
    };
    let actual = Duration::from_millis((backoff.as_millis() as i64 + jitter).max(0) as u64);
    tokio::select! {
        _ = tokio::time::sleep(actual) => true,
        _ = cancel.cancelled() => false,
    }
}
```

**IMPORTANT:** The backoff sleep must be cancellable. Use `tokio::select!` with `cancel.cancelled()` so that a shutdown signal during backoff does not delay drain.

**Metrics aggregator function:**

```rust
async fn metrics_aggregator(
    mut sample_rx: mpsc::Receiver<RequestSample>,
    snapshot_tx: watch::Sender<MetricsSnapshot>,
    cancel: CancellationToken,
    expected_interval_ms: u64,
    ramp_up_end: Instant,
) {
    // Dual recorder: live (all samples) and report (post-ramp-up only)
    let mut live = MetricsRecorder::new(expected_interval_ms);
    let mut report = MetricsRecorder::new(expected_interval_ms);
    let mut tick = tokio::time::interval(Duration::from_secs(2));
    tick.set_missed_tick_behavior(tokio::time::MissedTickBehavior::Delay);

    loop {
        tokio::select! {
            biased;  // Tick gets priority to prevent display starvation

            _ = tick.tick() => {
                // Drain all available samples before publishing snapshot
                while let Ok(sample) = sample_rx.try_recv() {
                    live.record(&sample);
                    if sample.timestamp >= ramp_up_end {
                        report.record(&sample);
                    }
                }
                let _ = snapshot_tx.send(live.snapshot());
            }
            result = sample_rx.recv() => {
                match result {
                    Some(sample) => {
                        live.record(&sample);
                        if sample.timestamp >= ramp_up_end {
                            report.record(&sample);
                        }
                    }
                    None => {
                        // All senders dropped -- VUs are done
                        let _ = snapshot_tx.send(report.snapshot());
                        break;
                    }
                }
            }
            _ = cancel.cancelled() => {
                // Drain remaining samples
                while let Ok(sample) = sample_rx.try_recv() {
                    live.record(&sample);
                    if sample.timestamp >= ramp_up_end {
                        report.record(&sample);
                    }
                }
                let _ = snapshot_tx.send(report.snapshot());
                break;
            }
        }
    }
}
```

**CRITICAL per Research Pitfall 2:** The `biased;` directive ensures the tick branch gets checked first, preventing display starvation when the mpsc channel is busy.

**CRITICAL per Research Pitfall 6:** Dual recorder approach -- `live` records ALL samples (for display), `report` records only post-ramp-up (for final result). The final `snapshot_tx.send()` publishes the `report` snapshot.
