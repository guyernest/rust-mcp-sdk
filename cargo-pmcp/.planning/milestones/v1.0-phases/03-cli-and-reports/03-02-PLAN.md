<objective>
Create the k6-style colorized terminal summary renderer as a pure function and wire it into the `loadtest run` command output.

Purpose: The terminal summary is the primary output a developer sees after running a load test. Without it, the run command only shows raw numbers. The k6-style format (dotted metric rows, ASCII header, color-coded values, error classification breakdown) gives developers an immediately familiar, professional view of their server's performance characteristics.

Output: `src/loadtest/summary.rs` (pure render function), updated `src/loadtest/metrics.rs` (error category tracking), updated `src/commands/loadtest/run.rs` (wiring).
</objective>

**Implementation details:**

1. **ASCII art header:**
```rust
fn render_header(url: &str, vus: u32, duration_secs: u64, scenario_count: usize) -> String {
    let header_art = format!(
        r#"
          /\      |  {}
         /  \     |
    /\  /    \    |  target:    {}
   /  \/      \   |  vus:       {}
  /    \       \  |  duration:  {}s
 /      \       \ |  scenarios: {} steps
"#,
        "cargo-pmcp loadtest".bold(),
        url,
        vus,
        duration_secs,
        scenario_count,
    );
    header_art
}
```

2. **Dotted metric row formatter:**
```rust
/// Format a single metric row with dot-padding.
///
/// Produces: "  metric_name..................: value_string"
fn format_metric_row(name: &str, value: &str, pad_width: usize) -> String {
    format!("  {name:.<pad_width$}: {value}")
}
```

Use a constant pad width of 40 for visual consistency.

3. **Metric rows:**
- `mcp_req_duration` -- show p50, p95, p99 (color green if p99 < 1000ms, yellow otherwise)
- `mcp_req_success_count` -- green
- `mcp_req_error_count` -- red if > 0, otherwise default
- `mcp_req_error_rate` -- red if > 5%, yellow if > 1%, green otherwise
- `mcp_req_throughput` -- compute as total_requests / elapsed_secs, green
- `mcp_req_total` -- total requests count

4. **Error breakdown section:**
Only show if there are errors. Iterate `error_category_counts` sorted by count descending:
```rust
if !snap.error_category_counts.is_empty() {
    lines.push(String::new());
    lines.push("  errors:".to_string());
    let mut categories: Vec<_> = snap.error_category_counts.iter().collect();
    categories.sort_by(|a, b| b.1.cmp(a.1));
    for (category, count) in categories {
        let row = format_metric_row(
            &format!("    {}", category),
            &count.to_string().red().to_string(),
            40,
        );
        lines.push(row);
    }
}
```

5. **Color coding rules (from CONTEXT.md):**
- Green for passing metrics (success count, throughput, low latency)
- Red for errors (error count > 0, high error rate)
- Yellow for warnings (p99 > 1000ms, error rate between 1-5%)

6. **Elapsed time line:**
```rust
lines.push(format_metric_row(
    "mcp_req_elapsed",
    &format!("{:.1}s", result.elapsed.as_secs_f64()),
    40,
));
```

**Full render_summary implementation:**

Assemble the header and metric lines into a single String joined by newlines. The function is deterministic given the same inputs (except for color escapes which depend on the global color override state).

    use crate::loadtest::config::{LoadTestConfig, ScenarioStep, Settings};

    use crate::loadtest::metrics::MetricsSnapshot;

        LoadTestConfig {
            settings: Settings {
                virtual_users: 10,
                duration_secs: 60,
                timeout_ms: 5000,
                expected_interval_ms: 100,

            scenario: vec![ScenarioStep::ToolCall {
                weight: 100,
                tool: "echo".to_string(),
                arguments: serde_json::json!({"text": "hello"}),
            }],
        }
    }

    fn success_snapshot() -> MetricsSnapshot {
        MetricsSnapshot {
            p50: 42,
            p95: 120,
            p99: 350,
            error_p50: 0,
            error_p95: 0,
            error_p99: 0,
            success_count: 950,
            error_count: 50,
            total_requests: 1000,
            error_rate: 0.05,
            operation_counts: HashMap::new(),
            per_operation_errors: HashMap::new(),
            error_category_counts: HashMap::from([
                ("timeout".to_string(), 30),
                ("jsonrpc".to_string(), 15),
                ("http".to_string(), 5),
            ]),
        }
    }

    #[test]
    fn test_render_summary_contains_header() {
        setup_no_color();
        let result = LoadTestResult {
            snapshot: success_snapshot(),
            elapsed: Duration::from_secs(60),
            final_active_vus: 10,

        assert!(output.contains("cargo-pmcp loadtest"), "Missing header title");
        assert!(output.contains("http://localhost:3000/mcp"), "Missing target URL");
        assert!(output.contains("10"), "Missing VU count");
    }

    #[test]
    fn test_render_summary_contains_latency_metrics() {
        setup_no_color();
        let result = LoadTestResult {
            snapshot: success_snapshot(),
            elapsed: Duration::from_secs(60),
            final_active_vus: 10,

        let config = minimal_config();
        let output = render_summary(&result, &config, "http://localhost:3000/mcp");

        assert!(output.contains("p50=42ms"), "Missing p50");
        assert!(output.contains("p95=120ms"), "Missing p95");
        assert!(output.contains("p99=350ms"), "Missing p99");
