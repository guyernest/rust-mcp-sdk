<objective>
Fix the `cargo pmcp deploy` root-resolution regression. `execute_async` currently resolves `project_root` via `find_workspace_root()` (workspace.rs:13), which walks up to the first ancestor with ANY `Cargo.toml`. In a `multi-crate-isolated` server dir nested inside a monorepo whose root has a `Cargo.toml`, the walk climbs PAST the dir holding `.pmcp/deploy.toml` to the monorepo root. Result: `init` scaffolds into the wrong dir; `deploy` then fails "Deployment not initialized".

Anchor root resolution on `.pmcp/deploy.toml` instead of `Cargo.toml`, restore 0.6.x-style cwd fallback for fresh `init`, add an explicit `--manifest-path` override, and add a Jidoka footgun guard so `init` never silently scaffolds into an unrelated ancestor.

Purpose: regression fix; restores correct deploy behavior in monorepo/isolated-server layouts.
Output: `find_deploy_root()` in workspace.rs; rewired `execute_async` resolution + new `--manifest-path` clap flag + init guard in deploy/mod.rs; full tempdir unit-test suite.
</objective>

```rust
/// Resolve a project root from an explicit --manifest-path override.
/// Accepts a dir (use <dir>) or a deploy.toml file (use its parent-of-.pmcp).
/// Errors when the override does not resolve to an existing .pmcp/deploy.toml.
fn resolve_manifest_override(path: &Path) -> Result<PathBuf> {
    // If it's a file named deploy.toml under a .pmcp dir → root is .pmcp's parent.
    if path.is_file() {
        let pmcp_dir = path.parent()
            .filter(|p| p.file_name().map_or(false, |n| n == ".pmcp"))
            .ok_or_else(|| anyhow::anyhow!(
                "--manifest-path file must be a .pmcp/deploy.toml: {}", path.display()))?;
        let root = pmcp_dir.parent().ok_or_else(|| anyhow::anyhow!(
            "--manifest-path .pmcp dir has no parent: {}", path.display()))?;
        if !root.join(".pmcp/deploy.toml").exists() {
            bail!("No .pmcp/deploy.toml at resolved root: {}", root.display());
        }
        return Ok(root.to_path_buf());
    }
    // Otherwise treat as a directory.
    if path.join(".pmcp/deploy.toml").exists() {
        return Ok(path.to_path_buf());
    }
    bail!("--manifest-path does not contain .pmcp/deploy.toml: {}", path.display());
}

/// Resolve project_root honoring precedence: --manifest-path override first,
/// then deploy-config anchoring. `for_init` controls the fallback:
///   - init  → find_deploy_root() Some, else current_dir() (0.6.x behavior)
///   - read  → find_deploy_root() Some, else "Deployment not initialized" error
fn resolve_project_root(&self, for_init: bool) -> Result<PathBuf> {
    if let Some(mp) = &self.manifest_path {
        return Self::resolve_manifest_override(mp);
    }
    match find_deploy_root()? {
        Some(root) => Ok(root),
        None if for_init => std::env::current_dir().context("Failed to get current directory"),
        None => bail!("Deployment not initialized. Run: cargo pmcp deploy init"),
    }
}

/// Jidoka footgun guard for init: refuse to scaffold into a resolved root
/// that is neither cwd nor an existing deploy root.
fn guard_init_root(root: &Path) -> Result<()> {
    let cwd = std::env::current_dir().context("Failed to get current directory")?;
    let same_as_cwd = root.canonicalize().ok() == cwd.canonicalize().ok();
    if !same_as_cwd && !root.join(".pmcp/deploy.toml").exists() {
        bail!(
            "Refusing to scaffold deployment into '{}': it is not the current directory \
             and has no existing .pmcp/deploy.toml. Run `cargo pmcp deploy init` from your \
             deploy directory, or pass --manifest-path <dir-or-deploy.toml>.",
            root.display()
        );
    }
    Ok(())
}
```
