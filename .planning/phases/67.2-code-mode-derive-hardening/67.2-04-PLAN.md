<objective>
Optimize eval.rs array methods to use scope-chain push/pop instead of cloning the entire local_vars HashMap per array element (P-01 from IMPROVEMENTS.md).

Purpose: Every array method (.map, .filter, .find, .some, .every, .flatMap, .reduce) currently clones the entire `local_vars` HashMap per element. For `.map()` over 100 items with 10 scope variables, that is 100 full HashMap clones. The push/pop pattern inserts the loop variable, evaluates, then restores the previous value -- one insert+remove per element instead of one clone.

Output: Modified eval.rs with scope-chain optimization. All array tests must produce identical results.

Per D-03: This change is behind the `js-runtime` feature gate (eval.rs is only compiled with that feature).
</objective>

**Strategy:** Change `evaluate_array_method_with_scope` to take `local_vars: &mut HashMap<String, JsonValue>`. The caller (`evaluate_with_scope`) will clone its local_vars once before calling the array method, then pass `&mut` to the clone.     // Regression test: verifies push/pop optimization handles large arrays
    // with many scope variables without excessive allocation.
    // Before optimization: 1000 HashMap clones (one per element).
    // After optimization: 1 HashMap clone (at array method entry).
    use std::time::Instant;

        .collect();
    let arr_value = JsonValue::Array(arr);

    // Simple map: x => x (identity)
    let method = ArrayMethodCall::Map {
        item_var: "x".to_string(),

    };
