<objective>
Wire policy evaluator into the derive macro's generated ValidateCodeHandler and switch from sync to async validation path.

Purpose: The current derive macro generates a handler that calls `validate_graphql_query()` (sync), which completely skips policy evaluation. This means the security contract (validate -> policy check -> token) is broken -- tokens are issued without any authorization check. This is the #1 blocker for pmcp.run team production deployment.

Output: Modified validation.rs with Arc-based policy evaluator, modified derive macro calling validate_graphql_query_async with the policy evaluator wired through the shared pipeline.
</objective>
