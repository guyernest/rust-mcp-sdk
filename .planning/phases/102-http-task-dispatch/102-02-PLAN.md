<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → Server::handle_request | Untrusted JSON-RPC over real HTTP now reaches the task lifecycle for the FIRST time (previously rejected). The new path must enforce the same auth + owner-scoping the ServerCore path already does. |
| tools/call (task field) → create-path | A client-sent `task` field + tool-fabricated taskId crosses into the create-path; the store must mint the canonical id. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-102-04 | Elevation of Privilege | Server::handle_request new tasks/* path | mitigate | Reuse Server's EXISTING auth validation (`auth_provider.validate_request`, mod.rs:1226-1236 on the call path); the shared `route_tasks_endpoint` receives an already-resolved `AuthContext` and never relaxes it. The new HTTP path must not bypass auth that the reject previously short-circuited. |
| T-102-05 | Information Disclosure | route_tasks_endpoint owner-scoping over HTTP | mitigate | Owner derives from `resolve_owner(AuthContext)` only; `store.get/list/cancel` enforce owner match (NotFound on mismatch). A second HTTP caller cannot read/cancel another caller's tasks (IDOR). Covered by the shared unit lifted in Plan 01; this plan must not pass a client-supplied owner. |
| T-102-06 | Tampering | create-path gate in handle_call_tool | mitigate | `maybe_build_task_created` mints the store id; the wire `task.taskId`/`_meta.relatedTask.taskId` are the store id, never the tool-fabricated one. The gate must also NOT emit a CreateTaskResult for a non-task tool call (proptest_task_branch_gate + server_call_tool_non_task assert this). |
| T-102-07 | Information Disclosure | pending tasks/result over HTTP | accept (frozen) | `-32002` "not completed" is the specified, frozen pending error; delegating to the shared unit preserves it. |
| T-102-SC | Tampering | npm/pip/cargo installs | mitigate | Zero external packages installed (research §Package Legitimacy Audit: not applicable). No `cargo add`. |
</threat_model>
