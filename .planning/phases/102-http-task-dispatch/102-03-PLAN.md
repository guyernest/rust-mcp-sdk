the HTTP path rejected `tasks/*`. With Plan 02 done, this plan replaces that shim
with a real HTTP round-trip — the carried-from-Phase-101 rule that protocol-shape

proof that pmcp.run can drop its `ServerCore::handle_request` shim.

fallback port), and BOTH the test and the example MUST abort/cancel the spawned
server task after the client completes (a spawned HTTP server can hang the process),
preferring readiness signaling over a fixed sleep where the harness allows.
