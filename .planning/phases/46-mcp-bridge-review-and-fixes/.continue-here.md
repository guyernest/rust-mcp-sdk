<decisions_made>

- Replaced Zod-pin URL rewriting (`fix_esm_cdn_imports`) with full inline shim (`inline_ext_apps_shim`) because Claude Desktop's Electron iframe blocks ALL external script loading, not just Zod-incompatible versions. The Zod pin was necessary but not sufficient.
- Shim implements core ext-apps protocol: `ui/initialize` handshake, `ui/toolResult` notifications, `tools/call` proxy. Covers 99% of widget use cases.
- Used import statement replacement (find `import...from "CDN_URL"`, replace with inline class definition) rather than import maps or data: URIs for maximum compatibility.
- Kept mcp-preview's separate import-map approach (redirects to local `widget-runtime.mjs`) since mcp-preview has a full bundled runtime available.

</decisions_made>
