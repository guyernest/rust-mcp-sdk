<objective>
Fix the bridge protocol method name mismatch that prevents tool result data from reaching widgets in Claude Desktop and standard MCP Apps hosts.

Purpose: The injected bridge scripts in adapter.rs listen for `ui/notifications/tool-result` (long form), but the ext-apps SDK and AppBridge send `ui/toolResult` (short form). This mismatch is the primary suspect for Claude Desktop widgets loading but never receiving data. Also, the McpApps adapter bridge lacks an `onToolResult` callback API -- it only dispatches raw `mcpNotification` events, which is not the developer-friendly API promised by the user decision ("widget developers write host-agnostic code using mcpBridge.onToolResult(callback)").

Output: Fixed adapter.rs bridge scripts, App class with method name normalization, confirmed AppBridge sends correct short-form.
</objective>
