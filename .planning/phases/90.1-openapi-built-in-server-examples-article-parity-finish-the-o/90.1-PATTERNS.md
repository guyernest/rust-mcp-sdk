#### Pattern B ŌĆö three `[[resources]]` blocks (analog reference-config.toml:269-415, 429+, 661-670 region)

The SQL reference ships resources as inline markdown addressed by URI. The
`[[resources]]` block shape (config.rs:765-785 ŌĆö `deny_unknown_fields`):
```toml
[[resources]]
uri = "docs://chinook/schema"
name = "Chinook Database Schema"
description = "Tables, columns, relationships for the Chinook music database"
mime_type = "text/markdown"
content = """
# Chinook Database Schema (SQLite)
...table reference, sample IDs, JOIN conditions...
"""
```
`ResourceDecl` fields: `uri` (String, required), `name` (Option<String>),
`description` (Option<String>), `mime_type` (Option<String>), `content` (Option<String>).

Mirror STRUCTURE onto TfL (RESEARCH.md:163-181, 247-249 ŌĆö do NOT copy SQL content;
there is no `[database]`/SQL in the OpenAPI world). Three TfL resources:
```toml
[[resources]]
uri = "docs://london-tube/schema"
name = "TfL Line API Schema"
description = "Endpoints, response shapes, and line ids for the TfL Line API"
mime_type = "text/markdown"
content = """
# TfL Line API (subset)
...endpoints (GET /Line/Mode/tube/Status, GET /Line/{lineId}/Disruption),
statusSeverity scale (severity < 10 = disrupted), sample line ids
(victoria, central)...
"""

[[resources]]
uri = "docs://london-tube/examples"
name = "TfL Example Scripts"
description = "Example Code Mode scripts for common TfL queries"
mime_type = "text/markdown"
content = """
# TfL Example Scripts
...api.get('/Line/Mode/tube/Status'); filter statusSeverity < 10;
fan-out to /Line/${id}/Disruption...
"""

[[resources]]
uri = "code-mode://learnings"
name = "TfL Code Mode Learnings"
description = "Tips and gotchas for authoring TfL Code Mode scripts"
mime_type = "text/markdown"
content = """
# TfL Code Mode Learnings
...bind api.get result to a const before return; line ids are lowercase;
maxLines bound from args...
"""
```

#### Pattern C ŌĆö one `[[prompts]]` block (analog reference-config.toml:661-670)

SQL gold-standard:
```toml
[[prompts]]
name = "start_code_mode"
description = "Load all context needed for Code Mode SQL generation"
include_resources = [
    "docs://chinook/schema",
    "code-mode://instructions",
    "code-mode://policies",
    "docs://chinook/examples",
    "code-mode://learnings"
]
```
`PromptDecl` fields (RESEARCH.md:196-197, config.rs:730-743): `name` (String),
`description` (Option<String>), `include_resources` (Vec<String>),
`arguments` (Vec<PromptArgumentDecl>).

TfL target (RESEARCH.md:185-195) ŌĆö **`include_resources` URIs must EXACTLY match the
`[[resources]]` `uri` values defined above** (Pitfall 2 / assemble.rs:204-213 resolves
them against the StaticResourceHandler; a typo points at a missing resource):
```toml
[[prompts]]
name = "start_code_mode"
description = "Load all context needed for Code Mode script generation"
include_resources = [
    "docs://london-tube/schema",
    "docs://london-tube/examples",
    "code-mode://learnings",
]
```

## Shared Patterns

### `deny_unknown_fields` strictness (applies to: london-tube.toml, example CONFIG, both chapters)
**Source:** `config.rs:101, 766` (every config section). RESEARCH.md:270-275, Pitfall 1.
Every `[[resources]]`/`[[prompts]]`/`[tools.annotations]` key MUST be one of the documented
fields. A misspelled key fails `from_toml_strict_validated` with `ToolkitError::Parse`.
Validate immediately after editing:
```sh
cargo test -p pmcp-openapi-server --test parity_replay london_tube_fixture
```

### `include_resources` URI integrity (applies to: london-tube.toml, example CONFIG, both chapters)
**Source:** `assemble.rs:204-213` (register_prompts resolves URIs against StaticResourceHandler).
RESEARCH.md:198-200, 277-281, Pitfall 2. Keep `[[prompts]].include_resources` in EXACT sync
with the defined `[[resources]].uri` values. A typo resolves to a missing/partial body.

### Offline + single-threaded replay discipline (applies to: scenarios.yaml, parity_replay.rs)
**Source:** parity_replay.rs:25-28, RESEARCH.md:289-293, Pitfall 4. Resources are STATIC inline
content ŌĆö assert via `list_resources`/`list_prompts` with NO backend round-trip / no new
wiremock mounts. Run single-threaded (ephemeral port + per-process `TFL_APP_KEY` env):
```sh
cargo test -p pmcp-openapi-server --test parity_replay -- --test-threads=1
```

### Build-only example safety (applies to: london_tube_min.rs)
**Source:** openapi_server_min.rs:5-10. `main` builds the server and returns WITHOUT serving ŌĆö
`cargo build --example` is the acceptance gate; never block on a live listener.

### Two-chapter sync (applies to: both .md files)
**Source:** RESEARCH.md:390-391 (both chapters grepped for "Resources"). Reuse the SAME TfL
`[[resources]]`/`[[prompts]]` TOML excerpts in book + course so they stay parity-aligned.
