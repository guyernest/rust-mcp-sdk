<objective>
Close DSTK-01: stop `cargo pmcp deploy` from silently overwriting an operator-curated `deploy/lib/stack.ts` on BOTH deploy targets, gated by an explicit `--regenerate-stack` (alias `--force`) opt-in.

Both write sites do an unconditional `std::fs::write(deploy/lib/stack.ts, …)`. This plan adds a `Path::exists()` guard that skips the write (but NOT the IAM validation) unless `config.regenerate_stack` is true, prints a "preserved existing stack.ts" notice, and wires the `--regenerate-stack`/`--force` flag from clap into `config.regenerate_stack`.

Purpose: Prevent silent data loss of curated stack.ts; make regeneration explicit per CONTEXT.md guard-behavior decisions (reporter candidates a + d).
Output: clap flag (mod.rs) + exists-guard on both targets (deploy.rs, targets/pmcp_run/deploy.rs); the RED guard test from 98-01 turns GREEN.
</objective>
