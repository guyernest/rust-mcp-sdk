<objective>
Close DSTK-02 + DSTK-03: make curated template metadata (`mcp:serverType`, `mcp:snapshotBaked`) reproducible-from-config so it survives a regeneration.

Today the render/synth path threads only IAM. `mcp:serverType` is hardcoded `'custom'` for pmcp.toml/custom servers, and `mcp:snapshotBaked` has zero representation anywhere. This plan threads the `[metadata]` block (added in 98-01) into `McpMetadata` (`server_type` override + new `snapshot_baked`), emits `mcp:snapshotBaked` from `to_cdk_context`, and teaches the stack template to read `mcp:snapshotBaked` from CDK context — mirroring exactly how `mcp:serverType` already flows.

Purpose: A regeneration becomes SAFE because curated literals can be reproduced from config (CONTEXT.md reporter candidate c). Backward-compatible: absent `[metadata]` ⇒ output unchanged.
Output: McpMetadata.snapshot_baked + server_type override (metadata.rs), template + render threading (init.rs), config-merge at the extraction site (targets/pmcp_run/deploy.rs); the 98-01 config-survives-render RED test turns GREEN.
</objective>
