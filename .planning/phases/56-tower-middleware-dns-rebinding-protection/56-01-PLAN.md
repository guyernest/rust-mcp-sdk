<objective>
Create the Tower Layer middleware stack for MCP server hosting: AllowedOrigins config (single source of truth for Host/Origin/CORS validation), DnsRebindingLayer (Host + Origin header validation per MCP spec CVE-2025-66414), and SecurityHeadersLayer (response hardening with OWASP defaults).

Purpose: Fix the CVE-pattern wildcard CORS (`Access-Control-Allow-Origin: *`) vulnerability and achieve MCP spec 2025-03-26 compliance for Origin validation. These layers are building blocks consumed by Plan 02's `pmcp::axum::router()`.

Output: Three new source files in `src/server/tower_layers/` with comprehensive unit tests.
</objective>
