Define `router()` and `router_with_config()`:
```rust
/// Create a secure MCP Axum Router with default localhost protection.
///
/// Returns an `axum::Router` with:
/// - DNS rebinding protection (Host + Origin validation)
/// - Security response headers (nosniff, DENY, no-store)
/// - Origin-locked CORS (no wildcard *)
///
/// Bind a listener and serve:
/// ```rust,no_run
/// # async fn example(app: axum::Router) {
/// let listener = tokio::net::TcpListener::bind("127.0.0.1:8080").await.unwrap();
/// axum::serve(listener, app).await.unwrap();
/// # }
/// ```
pub fn router(server: Arc<tokio::sync::Mutex<Server>>) -> Router {
    router_with_config(server, RouterConfig::default())
}

/// Create a secure MCP Axum Router with explicit configuration.
///
/// Use this when deploying to production with specific allowed origins:
/// ```rust,no_run
/// # use pmcp::server::axum_router::*;
/// # fn example(server: std::sync::Arc<tokio::sync::Mutex<pmcp::Server>>) {
/// let app = router_with_config(server, RouterConfig {
///     allowed_origins: Some(AllowedOrigins::explicit(vec![
///         "https://myapp.example.com".to_string(),
///     ])),
///     ..Default::default()
/// });
/// # }
/// ```
pub fn router_with_config(
    server: Arc<tokio::sync::Mutex<Server>>,
    config: RouterConfig,
) -> Router {
    let allowed = config
        .allowed_origins
        .unwrap_or_else(AllowedOrigins::localhost);

    let state = make_server_state(server, config.server_config);
    let base_router = build_mcp_router(state);

    let cors = CorsLayer::new()
        .allow_origin(allowed.to_cors_allow_origin())
        .allow_methods([
            Method::GET,
            Method::POST,
            Method::DELETE,
            Method::OPTIONS,
        ])
        .allow_headers([
            "content-type".parse().unwrap(),
            "accept".parse().unwrap(),
            "mcp-session-id".parse().unwrap(),
            "mcp-protocol-version".parse().unwrap(),
            "last-event-id".parse().unwrap(),
        ])
        .expose_headers([
            "mcp-session-id".parse().unwrap(),
            "mcp-protocol-version".parse().unwrap(),
        ])
        .max_age(Duration::from_secs(86400));

    // Layer ordering: ServiceBuilder applies top-to-bottom.
    // Request flow: CorsLayer (outermost, handles preflight) -> DnsRebindingLayer -> SecurityHeadersLayer (innermost) -> handler
    // Using Router::layer() which applies bottom-to-top for requests:
    // Last .layer() call runs FIRST on requests.
    base_router
        .layer(config.security_headers)
        .layer(DnsRebindingLayer::new(allowed))
        .layer(cors)
}
```

/// Add CORS headers to a `HeaderMap`.
///
/// When `allowed_origins` is provided, uses origin-locked CORS.
/// When `None`, defaults to localhost aliases.
fn add_cors_headers_for(headers: &mut HeaderMap, request_origin: Option<&HeaderValue>, allowed: &AllowedOrigins) {
    // Reflect the request's Origin if it's in the allowed list
    if let Some(origin) = request_origin {
        if allowed.is_allowed_origin(origin) {
            headers.insert("Access-Control-Allow-Origin", origin.clone());

**IMPORTANT ALTERNATIVE approach (simpler, preferred):** Since `add_cors_headers` is called from many handlers that don't have access to the request Origin, the simplest fix is:
- Keep `add_cors_headers` but change the wildcard `*` to reflect allowed origins. Since handlers don't have the request's Origin header, the cleanest approach is to make `add_cors_headers` take the `config` and the request's Origin:
  - For handler functions that receive `headers: HeaderMap`, extract Origin from the request headers and pass it through.
  - This requires threading the request Origin to `add_cors_headers` at ~10 call sites.

**Simplest viable approach:** Change `add_cors_headers` to accept the `AllowedOrigins` from the state config and the request origin header. At each call site in the handlers, extract `req_headers.get(header::ORIGIN)` and pass it. Where the config has `allowed_origins: None`, use `AllowedOrigins::localhost()` as the fallback.

Actually -- looking at the 10+ call sites, the LEAST invasive approach that still fixes the CVE is:
1. Make `add_cors_headers` take `(headers: &mut HeaderMap, allowed_origins: &AllowedOrigins, request_origin: Option<&HeaderValue>)`.
2. If request_origin is Some and allowed, reflect it. If None or not allowed, don't add Allow-Origin header.
3. Thread the `AllowedOrigins` from `state.config.allowed_origins` (defaulting to `AllowedOrigins::localhost()`) at each call site.
4. Thread the request Origin header at each call site.

The handler functions already receive `headers: HeaderMap` (request headers) and `State(state): State<ServerState>`, so both are accessible.
