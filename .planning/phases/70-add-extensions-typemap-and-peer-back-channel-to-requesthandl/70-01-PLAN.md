## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| middleware → handler | Untrusted values crossing the request-scoped state container (`RequestHandlerExtra.extensions`) between middleware layers and the final handler. |
| log sink → operator | Debug-formatted output of `RequestHandlerExtra` landing in logs that operators read. |
| clone consumer → shared-state writer | Cloned `RequestHandlerExtra` instances (e.g., inside middleware chains) that share Extensions content with the origin. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-70-01 | Information Disclosure | `RequestHandlerExtra::fmt` (Debug impl, src/server/cancellation.rs:296-335) | mitigate | Extensions' own `Debug` prints type names only, not values (verified at docs.rs/http/src/extensions.rs L268-286). Task 1 adds `.field("extensions", &self.extensions)` to the existing redaction-aware Debug chain — sensitive values stored in Extensions will NOT appear in formatted output. Unit test `test_debug_extensions_prints_type_names_only` (Task 1) asserts this by inserting a marker string and grepping the Debug output to confirm absence. |
| T-70-03 | Elevation of Privilege | Values stored in Extensions that outlive the request scope | mitigate | `http::Extensions` requires `T: Clone + Send + Sync + 'static` which means stored values must already be clonable — they cannot smuggle non-'static references.
