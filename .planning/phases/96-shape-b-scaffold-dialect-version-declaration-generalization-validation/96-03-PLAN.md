<objective>
Resolve the two WBEX critical-path landmines BEFORE authoring the loan/quirk fixtures:

1. The .xlsx authoring gap (Pitfall 1): there is NO in-repo .xlsx generator — `tax-calc.xlsx` is a
   committed binary. Build a reusable `#[cfg(test)]` `rust_xlsxwriter` fixture author that sets
   `DocProperties` for genuine Excel identity (`<Application>Microsoft Excel</Application>` + an
   `<AppVersion>` + non-sentinel calcId -> `ProvenanceClass::ExcelTrusted`) and writes formulas WITH
   cached results (the reconcile oracle), so the loan workbook (Plan 04) and the quirk corpus (Plan 05)
   author their fixtures through one proven helper that rides the trusted-fixture override. Committed
   .xlsx files are produced through a NON-MUTATING, reproducible generator path (an `#[ignore]`d,
   env-gated generator test), NEVER a normal test that writes into the source tree.
2. The 1900-leap-year Open Question (Pitfall 3 / RESEARCH Open Question 1): no date-serial code exists
   and DATE() is not whitelisted. Spike a tiny probe fixture to determine whether the leap-year serial
   offset is expressible with whitelisted ops over `f64`, and record the disposition (with explicit
   WBEX-02 traceability) so Plan 05 has a decided path — WITHOUT adding date functions
   (deferred-boundary violation).

Purpose: De-risk the critical path so the two WBEX gates (Plans 04, 05) build on a proven authoring
helper (with a direct provenance assertion + a reproducible generation workflow) and a decided quirk
disposition rather than stalling.
Output: A `fixture_author.rs` `#[cfg(test)]` module, a leap-year probe fixture + spike findings doc.
</objective>
