<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg (crate name) -> filesystem | `cargo pmcp new --kind workbook-server <name>` writes files under a name-derived dir. |
| scaffolded crate -> served binary tree | The emitted Cargo.toml's feature set determines whether the reader/code-mode leak into the served tree. |
| cargo-pmcp package -> published crate (crates.io) | Embedded assets must live inside the package root to survive `cargo publish`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-96-04 | Tampering | crate name path traversal (`new --kind workbook-server ../evil`) | mitigate | `validate_crate_name` called FIRST in execute_workbook_server, BEFORE any fs::write — rejects `/`, `\`, `..`, leading digit (reuse, do not reimplement; new.rs:116-140). |
| T-96-05 | Elevation | code-mode/umya/SWC leaking into the served tree via toolkit defaults | mitigate | Emitted Cargo.toml uses `default-features = false, features = ["workbook-embedded","http"]` (T-95-06); a `make purity-check` over the served cone confirms absence; the emitted-Cargo.toml unit test asserts no `code-mode`. |
| T-96-06 | Tampering | carried embedded bundle drifting from the committed golden (boot-integrity break) | mitigate | Bundle-bytes golden test asserts byte-equality between the in-package embedded asset and crates/pmcp-server-toolkit/tests/fixtures/tax-calc@1.1.0 (Pitfall 5). |
| T-96-06b | Tampering | scaffold assets missing after `cargo publish` (broken scaffold for external users) | mitigate | Assets committed UNDER the cargo-pmcp package root + embedded via include_dir!/include_bytes!; a `cargo package --list` smoke asserts they ship. |
| T-96-SC | Tampering | npm/pip/cargo installs | mitigate | No new external packages (clap/tokio/include_dir already vetted; tempfile is a dev-dep) — no install task -> no legitimacy checkpoint. |
</threat_model>
