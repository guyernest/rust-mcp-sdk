<objective>
Add the WBDL-02 dialect-version declaration: a workbook self-declares the dialect version it
targets in a reserved `pmcp_dialect_version` named range, and the compiler validates that
declaration with a fail-closed semver-compatibility policy (D-04) while treating an absent
declaration as the baseline version (D-05). The supported/baseline version contract lives in the
`pmcp-workbook-dialect` crate (parallel to WHITELIST) and is bound to the human spec doc by a
drift-guard test (parallel to WBDL-01).

Purpose: Enable forward-compatible dialect evolution without abandoning the milestone's fail-closed
ethos, and keep the version contract from drifting between code and spec.
Output: A new `dialect_version.rs` accessor, version consts + drift guard in the dialect crate, a
spec-doc version-policy section, plus the ALWAYS fuzz + property coverage for the version-string parse
and the compat decision.
</objective>
