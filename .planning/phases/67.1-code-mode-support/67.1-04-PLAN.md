4. Create `crates/pmcp-code-mode-derive/src/lib.rs` implementing `#[derive(CodeMode)]`.

The macro expects a struct with these required fields (detected by name and type):
- `code_mode_config: CodeModeConfig` (or `Arc<CodeModeConfig>`)
- `token_secret: TokenSecret` (or `Arc<TokenSecret>`)
- `policy_evaluator: Arc<dyn PolicyEvaluator>` (or `Arc<P>` where P: PolicyEvaluator)
- `code_executor: Arc<dyn CodeExecutor>` (or `Arc<E>` where E: CodeExecutor)

The macro generates:
```rust
impl StructName {
    pub fn register_code_mode_tools(
        &self,
        builder: pmcp::server::builder::ServerCoreBuilder,
    ) -> pmcp::server::builder::ServerCoreBuilder {
        // Creates Arc-wrapped state structs for validate and execute handlers
        // Registers "validate_code" and "execute_code" tools on builder
        builder
            .tool("validate_code", validate_handler)
            .tool("execute_code", execute_handler)
    }
}

// Send + Sync compile-time assertion (per D-08):
const _: fn() = || {
    fn assert_send_sync<T: Send + Sync>() {}
    assert_send_sync::<StructName>();
};
```

Implementation approach using darling for attribute parsing:
- Use `darling::FromDeriveInput` to parse the struct
- Iterate over fields to find required names
- Emit compile error if required fields are missing
- Generated tool handlers use `pmcp_code_mode::async_trait` (per D-07) -- NOT `async_trait::async_trait` directly
- The validate handler creates a `ValidationPipeline` from config + token_secret
- The execute handler verifies the token then calls `code_executor.execute()`

Key generated code patterns:
- `#[pmcp_code_mode::async_trait]` on generated handler impls
- `std::sync::Arc::new(...)` for state wrapping
- `pmcp_code_mode::ValidationPipeline::from_token_secret(&self.code_mode_config, &self.token_secret)` to create the pipeline (Plan 02 adds this secure constructor -- do NOT use `expose_secret().to_vec()` in generated code; secret exposure stays inside the validation module)
- `pmcp::server::builder::ServerCoreBuilder` as the builder type (NOT `ServerBuilder`)
