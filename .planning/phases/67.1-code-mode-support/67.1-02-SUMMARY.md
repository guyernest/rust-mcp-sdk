## Decisions Made
- **HmacTokenGenerator::new() signature change:** Takes `TokenSecret` directly; `new_from_bytes()` wraps raw bytes for migration. This ensures callers are aware they are handling secret material.
- **from_token_secret() encapsulation:** The `ValidationPipeline::from_token_secret()` method calls `expose_secret().to_vec()` internally so the derive macro generated code (Plan 04) never calls `expose_secret()` directly.
- **NoopPolicyEvaluator test-only documentation:** WARNING heading in doc comment, module-level note, and name() returning "noop" for audit trail -- addresses review concern #4.
