## Decisions Made
- **pmcp::Error::Internal for generated errors:** The pmcp crate's Error enum has no `ToolError` variant. Generated handlers use `Error::Internal` for argument parsing, token verification, and execution errors.
- **TokenGenerator trait import as _:** The generated handler module needs `verify` and `verify_code` methods from the `TokenGenerator` trait. Importing as `_` brings the methods into scope without namespace pollution.
- **Separate ValidationPipeline per handler:** Each handler (validate and execute) gets its own pipeline instance. This avoids shared mutable state and simplifies the generated code since `ValidationPipeline` is not behind `Arc<RwLock<_>>`.
- **Lowercase module name:** The generated module `__code_mode_impl_myserver` uses lowercase to satisfy Rust's snake_case convention. `#[allow(non_snake_case)]` added as belt-and-suspenders.
