## Decisions Made
- **Separate file for CodeExecutor:** The existing `executor.rs` is behind `#[cfg(feature = "js-runtime")]`, but CodeExecutor must be available without any feature flags. Created `code_executor.rs` as an always-compiled module rather than putting the trait inline in `lib.rs` -- this keeps lib.rs as a re-export hub and allows the trait module to grow independently.
- **PlanExecutor blanket impl deferred:** `PlanExecutor::execute` takes `&mut self` (it mutates internal variables and API call tracking state). A blanket `impl CodeExecutor for PlanExecutor<E>` would require wrapping PlanExecutor in `Mutex`/`RwLock` to satisfy CodeExecutor's `&self` signature. This complexity is not worth the convenience -- external servers implement CodeExecutor directly.
