<decisions_made>

- Examples should use `McpAppsAdapter` (standard bridge) as the default adapter, not `ChatGptAdapter`
- Tools declare UI association via `with_ui()` on `TypedSyncTool`, emitting standard `{ "ui": { "resourceUri": "..." } }` metadata
- ChatGPT support is opt-in via `.with_host_layer(HostType::ChatGpt)` on the builder, which enriches tool `_meta` at build time
- Resource handlers return `meta: None` — standard hosts don't need resource-level _meta, and ChatGPT enrichment happens via host layer or preview-side
- Examples SHOULD include `.with_host_layer(HostType::ChatGpt)` by default since most developers will want ChatGPT compatibility, with a comment explaining it's optional
</decisions_made>
