<feature>
  <name>translate_placeholders: dialect-aware placeholder translation with binding-order preservation</name>
  <files>crates/pmcp-server-toolkit/src/sql/translate.rs</files>
  <behavior>
    Input/output expectations (every case below is a unit-test target):
    
    **Empty / no placeholder:**
    - `translate_placeholders("", Dialect::Postgres)` → `TranslatedSql { sql: "", ordered_params: [] }`
    - `translate_placeholders("SELECT 1", Dialect::MySql)` → `TranslatedSql { sql: "SELECT 1", ordered_params: [] }`
    
    **Single placeholder:**
    - `("SELECT :id FROM t", Dialect::Postgres)` → `{ "SELECT $1 FROM t", ["id"] }`
    - `("SELECT :id FROM t", Dialect::MySql)` → `{ "SELECT ? FROM t", ["id"] }`
    - `("SELECT :id FROM t", Dialect::Athena)` → `{ "SELECT ? FROM t", ["id"] }`
    - `("SELECT :id FROM t", Dialect::Sqlite)` → `{ "SELECT :id FROM t", ["id"] }` — identity SQL, but still records bind order
    
    **Repeated placeholder name:**
    - `("WHERE a = :a AND b = :b AND c = :a", Dialect::Postgres)` → `{ "WHERE a = $1 AND b = $2 AND c = $3", ["a","b","a"] }` — fresh positional index per appearance
    
    **String literal containing `:`:**
    - `("SELECT 'WHERE name = :foo' AS x", Dialect::Postgres)` → `{ "SELECT 'WHERE name = :foo' AS x", [] }` — `:foo` inside `'...'` is NOT translated
    - Doubled single-quote escape `''` is handled (still inside literal)
    - Double-quoted identifier `"col:name"` treated as literal-style for placeholder skipping (per RESEARCH §2.2 spec)
    
    **Line comment:**
    - `("SELECT 1 -- bind :id here", Dialect::Postgres)` → `{ "SELECT 1 -- bind :id here", [] }` — `:id` after `--` not translated, ends at `\n`
    
    **Block comment:**
    - `("SELECT /* :foo */ 1", Dialect::Postgres)` → `{ "SELECT /* :foo */ 1", [] }` — `:foo` not translated
    - Nested block comments `/* /* :foo */ */` tracked via depth counter
    
    **Postgres cast `::text`:**
    - `("SELECT 'foo'::text", Dialect::Postgres)` → `{ "SELECT 'foo'::text", [] }` — bare `:` followed by non-identifier char (here, another `:`) emitted verbatim
    
    **Unterminated literal (adversarial):**
    - `("SELECT 'unterminated :foo", Dialect::Postgres)` → does NOT panic; treats remainder as string-literal content; returns whatever was accumulated (acceptable per RESEARCH §2.4 invariant 5)
    
    **Property tests (turn GREEN here):**
    1. `idempotence_no_placeholders` — for `s` with no `:` chars, all 4 dialects produce `translate.sql == s`
    2. `bind_order_preserved` — for any `s`, the names in `ordered_params` appear in left-to-right textual order matching their positions in the input
    3. `postgres_positional_indexing` — for any `s`, `ordered_params.len()` equals the count of `$N` tokens in Postgres output, and indices are 1..=n with no gaps
    4. `sqlite_identity` — for any `s`, `Dialect::Sqlite` produces `translate.sql == s` (SQL byte-identical; only `ordered_params` differs)
    5. `no_panic_on_arbitrary_input` — for ANY `&str` (including non-UTF-8 boundaries via `arb_string` proptest generator), the function returns without panicking
  </behavior>
  <implementation>
    Replace the stub `translate_placeholders` body in `src/sql/translate.rs` with the `SqlWalker` state machine per Pattern G (split-helpers, each ≤cog 25):
    
    1. **Define `State` enum** privately in the module:
       `Normal`, `StringLiteral(char)`, `LineComment`, `BlockComment(usize /* depth */)`, `Placeholder`.
    
    2. **`SqlWalker<'a>` struct** with fields: `chars: std::iter::Peekable<std::str::Chars<'a>>` (or `CharIndices`), `state: State`, `out: String`, `order: Vec<String>`, `pg_index: usize`, `dialect: Dialect`, `pending_name: String` (current placeholder being read).
    
    3. **`fn new(sql: &'a str, dialect: Dialect) -> Self`** — initializes all fields.
    
    4. **`fn run(&mut self)`** — outer dispatch loop; while there's a char, dispatch to handler based on `self.state`. Cog target: ~10. On EOF, if `state == Placeholder` and `pending_name` non-empty, call `self.flush_placeholder()` (emit translated form).
    
    5. **`fn handle_normal(&mut self, c: char)`** — handles: `:` → switch to `Placeholder` + clear `pending_name`; `'` or `"` → switch to `StringLiteral(c)` + emit verbatim; `-` peek-next `-` → switch to `LineComment` + emit verbatim; `/` peek-next `*` → switch to `BlockComment(1)` + emit verbatim; otherwise emit verbatim. Cog target: ~12. Use `peek()` for the two-char lookahead.
    
    6. **`fn handle_placeholder(&mut self, c: char)`** — if `c` is `[A-Za-z0-9_]` → push to `pending_name`; else → call `flush_placeholder(c)` (emits the translated form THEN re-dispatches `c` via `handle_normal`). Cog ~6.
    
    7. **`fn flush_placeholder(&mut self)`** — handles three cases per RESEARCH §2 + spike main.rs:103-116:
       - `pending_name.is_empty()` → emit ":" verbatim (lone-`:` case like `::text`), reset state to Normal.
       - Otherwise → call `emit_placeholder(&self.pending_name)`, record name in `self.order`, reset state to Normal.
    
    8. **`fn emit_placeholder(&mut self, name: &str)`** — dispatch per dialect (spike main.rs:108-116). Cog ~6.
       - `Postgres`: `self.pg_index += 1; write!(self.out, "${}", self.pg_index).unwrap();`
       - `MySql | Athena`: `self.out.push('?');`
       - `Sqlite`: `write!(self.out, ":{}", name).unwrap();` (identity)
    
    9. **`fn handle_string(&mut self, c: char, q: char)`** — emit `c` verbatim. If `c == q`: peek next char; if next is also `q`, emit verbatim + consume (doubled-quote escape `''`); else switch back to `Normal`. Cog ~6.
    
    10. **`fn handle_line_comment(&mut self, c: char)`** — emit `c` verbatim. If `c == '\n'`: switch to `Normal`. Cog ~3.
    
    11. **`fn handle_block_comment(&mut self, c: char, depth: usize)`** — emit `c` verbatim. Track nesting: `*` then `/` → depth-1 (Normal at depth 0); `/` then `*` → depth+1. Cog ~5.
    
    12. **`fn into_translated(self) -> TranslatedSql`** — returns `TranslatedSql { sql: self.out, ordered_params: self.order }`. Cog ~1.

        fn idempotence_no_placeholders(s in "[A-Za-z0-9 _\\.,;\\(\\)=]*") {
            // s contains no `:` — translate is identity for all dialects.
            for d in [Dialect::Postgres, Dialect::MySql, Dialect::Athena, Dialect::Sqlite] {
                let t = translate_placeholders(&s, d);
                prop_assert_eq!(t.sql, s.clone());
                prop_assert!(t.ordered_params.is_empty());
            }
        }

            prop_assert_eq!(t.sql, s);
        }

        fn no_panic_on_arbitrary_input(s in any::<String>()) {

                let _ = translate_placeholders(&s, d);
            }
        }
        
        #[test]
        fn bind_order_preserved(names in proptest::collection::vec("[a-z]{1,5}", 1..=5)) {
            // Construct SQL with placeholders in order; assert ordered_params matches names.
            let sql = names.iter().map(|n| format!(":{n}")).collect::<Vec<_>>().join(", ");
            let t = translate_placeholders(&sql, Dialect::Postgres);
            prop_assert_eq!(t.ordered_params, names);
        }

            let sql = names.iter().map(|n| format!(":{n}")).collect::<Vec<_>>().join(", ");
            let t = translate_placeholders(&sql, Dialect::Postgres);
            for i in 1..=names.len() {
                prop_assert!(t.sql.contains(&format!("${i}")));
            }
        }
    }

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-84-02-01 | Tampering | SQL injection if translator mistakenly translates `:foo` inside a string literal | mitigate | `StringLiteral(quote)` state explicitly skips translation. Unit test asserts: `translate("SELECT 'WHERE name = :foo'", Postgres).ordered_params.is_empty()` |
