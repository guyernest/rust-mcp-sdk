<objective>
Extend the Phase 83 `SqlConnector` trait from its 2-method MVP (`dialect()` + `schema_text()`) to the 3-method shape required by CONN-01 by adding `execute(&self, sql, params) -> Result<Vec<serde_json::Value>, ConnectorError>` without a default body. Also extend the `#[non_exhaustive]` `ConnectorError` enum with four new variants (`Driver`, `Query`, `ParameterBind { name, reason }`, `Connection`) that the per-backend crates in Wave 2 emit. The phase 83 trait rustdoc already warns this change is coming.

Purpose: Lock the trait surface so Wave 2's three parallel per-backend crates implement against a stable contract. The `#[non_exhaustive]` enum makes the variant additions semver-clean for downstream consumers; the trait method addition is semver-breaking but pre-publish, per P83 D-05's `0.1.0 → 0.1.0` disposition (`cargo search pmcp-server-toolkit` returns 0 hits as of 2026-05-19).

Output: One modified file (`crates/pmcp-server-toolkit/src/sql/mod.rs`) with the extended trait, the four new error variants, doctests on the new method, and unit tests asserting variant `Display` output never leaks credentials.
</objective>
