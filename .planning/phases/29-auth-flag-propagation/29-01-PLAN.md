  <name>Task 1: Define AuthFlags, AuthMethod, and resolve() in flags.rs</name>
  <files>cargo-pmcp/src/commands/flags.rs</files>
  <behavior>
    - Test: AuthFlags with no auth fields set -> resolve() returns AuthMethod::None
    - Test: AuthFlags with api_key set -> resolve() returns AuthMethod::ApiKey(key)
    - Test: AuthFlags with oauth_client_id set -> resolve() returns AuthMethod::OAuth with correct fields
    - Test: AuthFlags with oauth_client_id and scopes -> scopes propagated, not default
    - Test: AuthFlags with oauth_client_id but no scopes -> defaults to ["openid"]
    - Test: AuthFlags with oauth_no_cache=true -> no_cache is true in OAuth variant
    - Test: clap parse with --api-key and --oauth-client-id simultaneously -> parse error (conflicts_with)
  </behavior>
  <action>
    Add to the bottom of `flags.rs` (after existing ServerFlags):

    1. Define `AuthMethod` enum with three variants:
       - `None` -- no auth configured
       - `ApiKey(String)` -- static bearer token
       - `OAuth { client_id: String, issuer: Option<String>, scopes: Vec<String>, no_cache: bool, redirect_port: u16 }`

    2. Define `AuthFlags` struct with `#[derive(Debug, Args)]`:
       - `api_key: Option<String>` -- `#[arg(long, env = "MCP_API_KEY", conflicts_with = "oauth_client_id")]`
       - `oauth_client_id: Option<String>` -- `#[arg(long, env = "MCP_OAUTH_CLIENT_ID")]`
       - `oauth_issuer: Option<String>` -- `#[arg(long, env = "MCP_OAUTH_ISSUER")]`
       - `oauth_scopes: Option<Vec<String>>` -- `#[arg(long, env = "MCP_OAUTH_SCOPES", value_delimiter = ',')]`
       - `oauth_no_cache: bool` -- `#[arg(long)]`
       - `oauth_redirect_port: u16` -- `#[arg(long, env = "MCP_OAUTH_REDIRECT_PORT", default_value = "8080")]`

    3. Implement `AuthFlags::resolve(&self) -> AuthMethod`:
       - If api_key is Some -> return ApiKey
       - If oauth_client_id is Some -> return OAuth (scopes default to vec!["openid"] if None)
       - Otherwise -> return None

    4. Add `#[cfg(test)] mod tests` at the bottom of flags.rs with tests covering the behavior above.
       For the clap conflict test, use `clap::Command::try_get_matches_from` on a test CLI struct that flattens AuthFlags.

    Use `/// rustdoc` comments on AuthFlags, AuthMethod, and resolve() matching the existing documentation style.
  </action>

  </verify>
