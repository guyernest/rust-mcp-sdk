```rust
#[derive(Debug, Args)]
pub struct AuthFlags {
    #[arg(long, env = "MCP_API_KEY", conflicts_with = "oauth_client_id")]
    pub api_key: Option<String>,
    #[arg(long, env = "MCP_OAUTH_CLIENT_ID")]
    pub oauth_client_id: Option<String>,
    #[arg(long, env = "MCP_OAUTH_ISSUER")]
    pub oauth_issuer: Option<String>,
    #[arg(long, env = "MCP_OAUTH_SCOPES", value_delimiter = ',')]
    pub oauth_scopes: Option<Vec<String>>,
    #[arg(long)]
    pub oauth_no_cache: bool,
    #[arg(long, env = "MCP_OAUTH_REDIRECT_PORT", default_value = "8080")]
    pub oauth_redirect_port: u16,
}

pub enum AuthMethod {
    None,
    ApiKey(String),
    OAuth { client_id: String, issuer: Option<String>, scopes: Vec<String>, no_cache: bool, redirect_port: u16 },
}

impl AuthFlags {
    pub fn resolve(&self) -> AuthMethod;
}
```

Since we cannot pass auth through that library function without modifying it, and modifying the library function is out of scope for this plan (it would touch mcp-tester crate and potentially break the standalone binary API), the approach is:
       - Resolve auth and log a warning if auth is configured but scenarios are being run through `run_scenario_with_transport()`: the library function does not support auth passthrough yet.
       - At the top of execute(), resolve auth: `let auth_method = auth_flags.resolve();`
       - If auth_method is not None, print a warning: "Warning: Auth flags are accepted but run_scenario_with_transport does not yet support auth passthrough. For authenticated servers, use `cargo pmcp test check` to verify connectivity."
       - This is the correct approach per the clean-break precedent. The full auth plumbing through the library will be addressed in Phase 30 when test commands are reworked via mcp-tester integration.
