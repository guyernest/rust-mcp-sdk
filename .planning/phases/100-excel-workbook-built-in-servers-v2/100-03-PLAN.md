<objective>
Perform the core engineering lift (WBV2-03, §4.1): extend the shared artifact model from
single-tool `CellMap { inputs[], outputs[] }` to multi-tool `CellMap { inputs[], tools[] }`,
where each `Tool` owns its own `outputs` + a DAG-derived `input_keys` set. Add the new
`Dag::upstream_input_leaves` reachability traversal (§4.2) and `build_tools` which groups
output cells by their owning Table and derives each tool's minimal input schema, and prove
the traversal total over hostile DAGs with both a property test and a fuzz target.

This plan touches ONLY the model + the pure DAG algorithm + the compiler-side emitter
(`build_tools`). The served-side fan-out (schema.rs / handler.rs / mod.rs) and the per-tool
reconcile re-key (`comparison_from_outputs`) are deliberately deferred to Plan 04 so that the
single→multi served fan-out (Pitfall 3) lands atomically in one consistent tree.

CROSS-WAVE COMPILE CONTRACT (resolves review finding #1): because the served call sites
(`schema.rs`/`handler.rs`/`mod.rs`) and the compiler reconcile still reference `outputs` until
Plan 04, this plan ships a TRANSITIONAL deprecated `CellMap::outputs()` accessor derived from
`tools[]` so the WHOLE workspace compiles green at the end of wave 3 — no multi-wave red
workspace. Plan 04 retires the shim once the served fan-out lands (so no dead compat code
survives — zero SATD). The plans stay SEPARATE (the design mandates the atomic fan-out is its
own plan); the shim — not a merge — is the sanctioned resolution.

Purpose: The `tools[]` model + `upstream_input_leaves` are the contract every served handler
and schema builder in Plan 04 consumes. Defining them first (interface-first) hands Plan 04 the
types instead of a scavenger hunt; the transitional accessor keeps the tree buildable between waves.

Output: New `Tool` type, `upstream_input_leaves`, `build_tools`, a transitional `outputs()`
accessor, + property test + fuzz target for reachability, with the workspace compiling green.
</objective>
