<objective>
Make ingest HARVEST Excel Tables (WBV2-02): today `table_ranges` reads only each table's
*area* and `SheetRecord.tables: Vec<RangeRef>` carries no name or column metadata (Pitfall 1).
Add a `TableRecord { name, area, columns }`, a `table_records()` ingest fn calling umya's
verified `Table::get_name()` / `get_columns()`, and per-row harvest of type/unit/enum/tier from
each row's `value` cell. Contain umya's malformed-table-XML `.unwrap()` (Pitfall 2 / §2 caveat 1)
behind the existing umya-isolation boundary as a clean `CompileError`, proven by a new fuzz target.

Purpose: The harvested Table name/columns/rows are the raw material the manifest model lift
(Plan 03) and multi-tool emission (Plan 04) consume. Without harvested names there is no tool
name and no per-row schema.

Output: Extended ingest model + harvest fn + per-row synth projection + a fuzz target.
</objective>
