<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM client → per-tool inputSchema | A client trusts the advertised schema; it must never be able to send a key the runtime then rejects, nor pin a computed/strict cell |
| output Table name → MCP tool name | An unsanitized/empty name would yield an uncallable or charset-illegal tool |
| BA-governed strict constant → tool input | A strict-tier field exposed as an input is a governance/privilege bypass |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-100-08 | Tampering | strict (BA-governed) constant supplied as a caller input | mitigate | tier column → InputTier::strict excluded from every tool's caller inputs; `is_strict_constant` reject (manifest_model.rs:205) + `strict_constant_override` error (error.rs:143) preserved; per-tool schema must not advertise a strict cell as an input |
| T-100-09 | Tampering | caller pins a computed output via override | mitigate | `is_computed` reject gate retained (manifest_model.rs:215); per-tool inputSchema only advertises DAG-derived Role::Input leaves (Plan 03 property) |
| T-100-10 | Spoofing/Tampering | unmappable/empty tool name → uncallable or charset-illegal tool | mitigate | `sanitize_tool_name` enforces `^[a-zA-Z0-9_-]{1,64}$` and rejects empty; an unmappable name is a cell-precise compile lint (fail-closed, never ships a bad tool) |
| T-100-11 | Information Disclosure | per-tool schema relaxing the strict envelope | mitigate | every per-tool inputSchema keeps `additionalProperties:false` mirroring the runtime `deny_unknown_fields` DTO (V5); F2 override block is the only sanctioned extra-key surface |
| T-100-SC | Tampering | npm/pip/cargo installs | accept | No package installs — all deps are existing workspace members |
</threat_model>
