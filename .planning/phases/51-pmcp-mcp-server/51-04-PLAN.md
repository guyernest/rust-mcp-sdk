1. Define `DocsResourceHandler` unit struct.
2. Implement `ResourceHandler::list()`:
   - Return `ListResourcesResult` with a `Vec<ResourceInfo>` containing all available documentation resources.
   - Each resource has: uri (e.g., "pmcp://docs/typed-tools"), name (e.g., "Typed Tools Guide"), description, mime_type "text/markdown".
   - Resources to list:
     - pmcp://docs/typed-tools
     - pmcp://docs/resources
     - pmcp://docs/prompts
     - pmcp://docs/auth
     - pmcp://docs/middleware
     - pmcp://docs/mcp-apps
     - pmcp://docs/error-handling
     - pmcp://docs/cli
     - pmcp://docs/best-practices
   - No pagination needed (< 20 resources), return next_cursor: None.

3. Implement `ResourceHandler::read()`:
   - Match on URI string to return the corresponding embedded content.
   - Return `ReadResourceResult` with a single `ResourceContents::text(uri, content, Some("text/markdown".to_string()))` entry.
   - Note: `ResourceContents` may need to be constructed manually as:
     ```rust
     ResourceContents {
         uri: uri.to_string(),
         mime_type: Some("text/markdown".to_string()),
         text: Some(content.to_string()),
         blob: None,
     }
     ```
     Check the actual `ResourceContents` struct definition and use the appropriate constructor.
   - For unknown URIs, return `Err(pmcp::Error::resource_not_found(uri))`.

**Workflow Prompts (prompts/workflows.rs):**

Create individual prompt handler structs for guided workflows. Each prompt implements `PromptHandler`.

Prompts to implement:

1. `QuickstartPrompt` -- no args. Returns a step-by-step guide to create a first PMCP server.
2. `CreateMcpServerPrompt` -- args: `name` (required), `template` (optional: minimal/calculator). Returns workspace setup instructions.
3. `AddToolPrompt` -- args: `tool_name` (required), `description` (optional). Returns how to add a new tool to an existing server.
4. `DiagnosePrompt` -- args: `server_url` (required). Returns diagnostic steps and suggests using test_check tool.
5. `SetupAuthPrompt` -- args: `auth_type` (optional: oauth/api-key/jwt). Returns auth configuration guide.
6. `DebugProtocolErrorPrompt` -- args: `error_message` (optional). Returns protocol debugging steps.
7. `MigratePrompt` -- no args. Returns guide for migrating from TypeScript SDK to PMCP.

Each prompt implementation:
- `handle()`: Build `GetPromptResult` with `messages` containing a User message (the question) and an Assistant message (the detailed answer). Use embedded content strings defined as `const &str` within the module.
- `metadata()`: Return `PromptInfo` with name, description, and argument list with `PromptArgument` entries marking required/optional.

Keep prompt response content concise and actionable -- 20-50 lines of guidance with code snippets. These are conversation starters for AI agents, not exhaustive documentation (the resources handle that).

Update `prompts/mod.rs`:
