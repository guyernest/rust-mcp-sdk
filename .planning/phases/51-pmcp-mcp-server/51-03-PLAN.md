<task type="auto">
  <name>Task 1: Implement scaffold tool with embedded templates</name>
  <files>crates/pmcp-server/src/tools/scaffold.rs</files>
  <action>
Create `scaffold.rs` implementing `ToolHandler` that returns code templates as structured JSON instructions. The tool does NOT write files -- it returns file paths and content for the AI agent or user to apply.

Input schema:
- `template` (string, required, enum: ["minimal", "calculator", "with-resources", "with-prompts", "mcp-app"]): Template type
- `name` (string, required): Project name (used in Cargo.toml, module names, etc.)

Implementation:

1. Define template content as `const &str` blocks within the module. Write concise, self-contained Rust templates that demonstrate PMCP SDK patterns. Each template should be a complete, compilable example. Key templates:

   - `minimal`: Workspace Cargo.toml + core lib.rs with one echo tool + server main.rs with StreamableHttpServer
   - `calculator`: Core lib.rs with add/subtract/multiply/divide typed tools + server main.rs
   - `with-resources`: Core lib with a tool + resource handler serving static content
   - `with-prompts`: Core lib with a tool + prompt handler with workflow template
   - `mcp-app`: Core lib with typed tool + with_ui() for MCP Apps widget

2. Define `ScaffoldTool` unit struct.

3. `handle()`:
   - Deserialize args to get template and name.
   - Replace `{name}` placeholders in template strings.
   - Return JSON with structure:
   ```json
   {
     "template": "minimal",
     "name": "my-server",
     "instructions": "Create a minimal MCP server workspace...",
     "files": [
       {"path": "Cargo.toml", "content": "...", "description": "Workspace manifest"},
       {"path": "crates/mcp-{name}-core/src/lib.rs", "content": "...", "description": "Core library"},
       {"path": "{name}-server/src/main.rs", "content": "...", "description": "Server binary"}
     ],
     "next_steps": [
       "Run `cargo build` to verify compilation",
       "Run `cargo run -p {name}-server` to start the server",
       "Run `cargo pmcp test check http://localhost:8080` to validate"
     ]
   }
   ```

4. `metadata()`: Return `ToolInfo` with name "scaffold", description, and the enum-based input schema.

Keep templates concise -- each file template should be 20-60 lines of Rust code. These are starting points, not production code. Use the patterns from the existing codebase (Server::builder(), ToolHandler trait, StreamableHttpServer).

IMPORTANT per user decision: "Build tools return instructions + code snippets (not file generation) -- AI agents or users apply them."

<task type="auto">
  <name>Task 2: Implement schema_export tool</name>
  <files>crates/pmcp-server/src/tools/schema_export.rs</files>
  <action>
Create `schema_export.rs` implementing `ToolHandler` that connects to a remote MCP server and exports its tool/resource/prompt schemas.

Input schema:
- `url` (string, required): MCP server URL to export schemas from
- `format` (string, optional, default "json", enum: ["json", "rust"]): Output format
- `timeout` (integer, optional, default 30): Timeout in seconds

Implementation:

1. Define `SchemaExportTool` unit struct.

2. `handle()`:
   - Deserialize args.
   - Create `ServerTester::new()` with URL.
   - Call `run_quick_test()` to initialize and discover capabilities.
   - Get tools via `get_tools()`, resources via `list_resources()`, prompts via `list_prompts()`.
   - Build export response:
     - `format == "json"`: Return the raw tools/resources/prompts arrays as JSON with their full schema information.
     - `format == "rust"`: Generate Rust type stubs from tool input schemas. For each tool, generate a `#[derive(Deserialize)] struct {ToolName}Input { ... }` with fields derived from the JSON Schema properties. This is a best-effort codegen -- string -> String, number -> f64, integer -> i64, boolean -> bool, array -> Vec<Value>, object -> Value.
   - Return JSON:
   ```json
   {
     "server_name": "...",
     "server_version": "...",
     "format": "json",
     "tools": [...],
     "resources": [...],
     "prompts": [...],
     "rust_types": "..." // only when format=rust
   }
   ```

3. `metadata()`: Return `ToolInfo` with name "schema_export", description, and input schema.

For Rust type generation: iterate tool input_schema properties, map JSON Schema types to Rust types, generate struct definition as a string. Include `#[serde(rename_all = "camelCase")]` if any field names contain camelCase. Wrap in a `rust_types` string field in the response.

After implementing, update `tools/mod.rs` to include:
```rust
pub mod scaffold;
pub mod schema_export;
pub use scaffold::ScaffoldTool;
pub use schema_export::SchemaExportTool;
```
(Add these to the existing mod.rs that already has test_check, test_generate, test_apps from Plan 02 -- or if Plan 02 hasn't run yet, include all 5 tool declarations.)
