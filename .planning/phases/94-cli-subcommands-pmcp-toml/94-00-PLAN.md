<objective>
Publish the THREE library seams the thin-shell `cargo pmcp workbook` CLI needs so the FULL gated
business-analyst (BA) flow ‚Äî version-from-the-workbook, gate-before-write on a re-compile, and an
ungated emit whose status travels with the artifact ‚Äî ships in Phase 94. Phase 93 left these verbs
PRIVATE; this plan makes them PUBLIC API on `pmcp-workbook-compiler` WITHOUT adding new compiler logic.
It surfaces/composes what already exists.

This plan is the Wave-0 library seam that unblocks 94-03 (gated compile lane) and 94-04 (hash-covered
ungated marker). It carries the WBCL IDs it unblocks: WBCL-01 (the gated half of compile) and WBCL-03
(the emit marker channel).

Purpose: Keep cargo-pmcp a THIN shell ‚Äî the compiler is the right home for (a) the workbook-version
accessor, (b) the gated-update candidate-build facade, and (c) the tamper-evident gate-marker channel.
All three are reusable by Phase 95/96 too.
Output: `crates/pmcp-workbook-compiler/src/version.rs` (new), edits to `lib.rs` (the `prepare_candidate`
facade + `Candidate` struct + re-exports) and `artifact/evidence.rs` + `artifact/mod.rs` (the marker
channel re-export).
