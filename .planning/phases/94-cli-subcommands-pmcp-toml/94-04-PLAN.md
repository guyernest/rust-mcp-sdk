<objective>
Implement the `workbook emit` handler (WBCL-03): regenerate a bundle WITHOUT the golden-corpus gate
for dev/reference, with a loud `UNGATED` banner AND a HASH-COVERED `gated: false` marker in the
bundle's `evidence/` (via the 94-00 `write_gate_marker` library channel) so an unvetted bundle can
never be mistaken for a promoted one downstream and a stripped/edited marker is detectable (D-08).

This is `compile.rs` minus the gate. It adds NO compiler logic — the marker channel and the
workbook-version reader are PUBLIC library API as of Plan 94-00.
Purpose: A fast dev/reference rebuild path whose tamper-evident ungated status TRAVELS with the artifact.
Output: `cargo-pmcp/src/commands/workbook/emit.rs` (replaces the Plan-02 neutral handler).

The prior blocking decision is resolved: the `gated: false` evidence-marker seam (`write_gate_marker`)
was added to `pmcp-workbook-compiler` in 94-00 as a hash-covered, tamper-evident channel that does NOT
mutate the served loader's frozen member set. The CLI consumes it directly — there is NO cli-side
loose-marker fallback (that weaker, non-hash-covered approach is dropped).
</objective>
