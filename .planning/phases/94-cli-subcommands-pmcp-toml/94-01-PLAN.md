<objective>
Add the `pmcp.toml` project-config parser (WBCL-04) and the cargo-pmcp ‚Üí pmcp-workbook-compiler
dependency edge. This is the foundation every workbook subcommand builds on: it kills the
lighthouse single-workbook justfile/path assumption by letting a project map many workbooks ‚Üí
bundle IDs in one file.

Purpose: Resolve a bundle-id (or "all declared workbooks") to a concrete (path, out_dir) without
the lighthouse's hardcoded reference path, and CONTAIN every resolved write target under the
project root. Optional-with-fallback (D-03): a bare-path invocation needs no toml at all.
Output: `cargo-pmcp/src/commands/workbook/config.rs` (new), `cargo-pmcp/Cargo.toml` (dep added).
</objective>

## Trust Boundaries

|----------|-------------|
| project `pmcp.toml` ‚Üí filesystem write paths | a checked-in (possibly adversarial/careless) config supplies `out_dir`/`path` that later subcommands write under |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-94-01-PATH | Tampering / Elevation | `WorkbookEntry.out_dir`/`path` resolved write target | mitigate | `validate(project_root)` rejects any `out_dir`/`path` that resolves OUTSIDE `project_root` ‚Äî a `..`-escaping RELATIVE path OR an ABSOLUTE path that does not `starts_with` the root. Containment is computed against the passed-in `project_root` (lexical `..`-rejection alone is insufficient ‚Äî concern C from the cross-AI review). Unit-tested AND fuzz-tested (Task 3 property #3 asserts neither a `../`-escaping nor an absolute out_dir is ever accepted). |
| T-94-01-DUP | Tampering | duplicate `bundle_id` shadowing | mitigate | `validate(project_root)` rejects duplicate bundle_ids so a later entry cannot silently shadow an earlier resolution. |
| T-94-01-TOML | Denial of Service | malformed toml | mitigate | `toml::from_str` errors are surfaced with context and fail the command cleanly; Task 3 property #1 fuzzes BOUNDED TOML bytes to prove the load path never panics (must-not-panic), bounding the malformed-input failure mode to a clean `Err`. toml is local, git-tracked, low-value ‚Äî no parser hardening beyond the crate default + the no-panic fuzz guard. |
</threat_model>
