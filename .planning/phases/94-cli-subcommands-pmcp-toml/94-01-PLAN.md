<objective>
Add the `pmcp.toml` project-config parser (WBCL-04) and the cargo-pmcp → pmcp-workbook-compiler
dependency edge. This is the foundation every workbook subcommand builds on: it kills the
lighthouse single-workbook justfile/path assumption by letting a project map many workbooks →
bundle IDs in one file.

Purpose: Resolve a bundle-id (or "all declared workbooks") to a concrete (path, out_dir) without
the lighthouse's hardcoded reference path. Optional-with-fallback (D-03): a bare-path invocation
needs no toml at all.
Output: `cargo-pmcp/src/commands/workbook/config.rs` (new), `cargo-pmcp/Cargo.toml` (dep added).
</objective>

- `pub fn validate(&self) -> Result<()>` — reject duplicate bundle_ids and `..`-escaping out_dir

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| project `pmcp.toml` → filesystem write paths | a checked-in (possibly adversarial/careless) config supplies `out_dir`/`path` that later subcommands write under |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-94-01-PATH | Tampering / Elevation | `WorkbookEntry.out_dir` resolved write target | mitigate | `validate()` rejects any `out_dir` whose normalized form escapes the project root via `..`; resolution joins under `project_root`. Unit-tested. |
| T-94-01-DUP | Tampering | duplicate `bundle_id` shadowing | mitigate | `validate()` rejects duplicate bundle_ids so a later entry cannot silently shadow an earlier resolution. |
| T-94-01-TOML | Denial of Service | malformed toml | accept | `toml::from_str` errors are surfaced with context and fail the command cleanly; toml is local, git-tracked, low-value — no parser hardening beyond the crate default. |
</threat_model>
