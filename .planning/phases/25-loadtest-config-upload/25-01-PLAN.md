<objective>
Implement the `cargo pmcp loadtest upload` command that validates a loadtest TOML config file locally and uploads it to pmcp.run for remote execution.

Purpose: Users need to push their loadtest configurations to pmcp.run for cloud-based load test execution. This mirrors the existing `cargo pmcp test upload` command but for loadtest TOML configs instead of test scenario YAML files.

Output: A working `cargo pmcp loadtest upload --server-id <id> <path>` command that validates the TOML, authenticates with pmcp.run, uploads via GraphQL, and displays success/error feedback.
</objective>

**Key differences from test upload:**
1. Loadtest upload accepts a single TOML file path, NOT multiple YAML paths
2. Loadtest upload VALIDATES the TOML via `LoadTestConfig::from_toml()` before uploading
3. The GraphQL mutation is `uploadLoadtestConfig` (new), NOT `uploadTestScenario` (existing)
4. Format is always "toml" (not yaml/json)
5. The config name defaults to the filename stem (e.g., "loadtest" from "loadtest.toml")

**GraphQL mutation design:** The new `uploadLoadtestConfig` mutation mirrors `uploadTestScenario` but with loadtest-specific fields:
```graphql
mutation UploadLoadtestConfig(
    $serverId: String!
    $name: String!
    $description: String
    $content: String!
    $format: String
) {
    uploadLoadtestConfig(
        serverId: $serverId
        name: $name
        description: $description
        content: $content
        format: $format
    ) {
        configId
        version
    }
}
```

```rust
pub async fn upload_loadtest_config(
    access_token: &str,
    server_id: &str,
    name: &str,
    description: Option<&str>,
    content: &str,
    format: &str,
) -> Result<UploadLoadtestConfigResult> {
    let query = r#"
        mutation UploadLoadtestConfig(
            $serverId: String!
            $name: String!
            $description: String
            $content: String!
            $format: String
        ) {
            uploadLoadtestConfig(
                serverId: $serverId
                name: $name
                description: $description
                content: $content
                format: $format
            ) {
                configId
                version
            }
        }
    "#;

    let variables = serde_json::json!({
        "serverId": server_id,
        "name": name,
        "description": description,
        "content": content,
        "format": format.to_lowercase()
    });

    #[derive(Debug, Deserialize)]
    struct UploadLoadtestConfigResponse {
        #[serde(rename = "uploadLoadtestConfig")]
        upload_loadtest_config: UploadLoadtestConfigResult,
    }

    let response: UploadLoadtestConfigResponse =
        execute_graphql(access_token, query, variables).await?;

    Ok(response.upload_loadtest_config)
}
```

```rust
//! Upload loadtest configs to pmcp.run

use anyhow::{Context, Result};
use colored::Colorize;
use std::path::PathBuf;

use crate::deployment::targets::pmcp_run::{auth, graphql};
use cargo_pmcp::loadtest::config::LoadTestConfig;

/// Upload a loadtest TOML config to pmcp.run for cloud execution.
pub async fn execute(
    server_id: String,
    path: PathBuf,
    name: Option<String>,
    description: Option<String>,
) -> Result<()> {
    println!(
        "\n{}",
        "Uploading loadtest config to pmcp.run".bright_cyan().bold()
    );
    println!(
        "{}",
        "─────────────────────────────────────────".bright_cyan()
    );

    // Step 1: Read the file
    let content = std::fs::read_to_string(&path)
        .with_context(|| format!("Failed to read config file: {}", path.display()))?;

    // Step 2: Validate TOML — parse and check scenarios exist
    //
    // This catches: missing file (above), invalid TOML syntax, missing [settings],
    // missing [[scenario]], zero total weight, invalid stage durations.
    // Uses the same validation as `cargo pmcp loadtest run`.
    if let Err(e) = LoadTestConfig::from_toml(&content) {
        // Provide actionable guidance based on error type
        eprintln!("\n  {} {}", "Error:".red().bold(), e);
        eprintln!();
        eprintln!("  {}", "The config file failed validation. To fix:".yellow());
        eprintln!(
            "    - Ensure the file is valid TOML syntax"
        );
        eprintln!(
            "    - Include a [settings] block with virtual_users, duration_secs, timeout_ms"
        );
        eprintln!(
            "    - Include at least one [[scenario]] block with type, weight, and operation fields"
        );
        eprintln!(
            "    - Run `cargo pmcp loadtest init` to generate a valid starter config"
        );
        anyhow::bail!("Config validation failed: {}", e);
    }

    // Step 3: Derive config name from filename if not provided
    let config_name = name.unwrap_or_else(|| {
        path.file_stem()
            .and_then(|s| s.to_str())
            .unwrap_or("loadtest")
            .to_string()
    });

    println!("  {} Config: {}", "→".blue(), path.display());
    println!("  {} Name: {}", "→".blue(), config_name);
    println!("  {} Server ID: {}", "→".blue(), server_id);
    println!("  {} Validated: config has valid scenarios", "✓".green());

    // Step 4: Authenticate with pmcp.run
    let credentials = auth::get_credentials().await?;

    // Step 5: Upload via GraphQL
    println!("\n  Uploading config...");

    match graphql::upload_loadtest_config(
        &credentials.access_token,
        &server_id,
        &config_name,
        description.as_deref(),
        &content,
        "toml",
    )
    .await
    {
        Ok(result) => {
            println!();
            println!(
                "{}",
                "═════════════════════════════════════════".bright_cyan()
            );
            println!(
                "{} Uploaded '{}' (config: {}, v{})",
                "✓".green().bold(),
                config_name,
                result.config_id,
                result.version
            );
            println!();
            println!("{}", "Next steps:".bright_white().bold());
            println!(
                "  - View config at: https://pmcp.run/servers/{}/loadtest",
                server_id
            );
            println!("  - Trigger a cloud load test from the pmcp.run dashboard");
            println!(
                "  - Or run locally: cargo pmcp loadtest run <url> --config {}",
                path.display()
            );
            println!(
                "{}",
                "═════════════════════════════════════════".bright_cyan()
            );
        },
        Err(e) => {
            eprintln!();
            eprintln!(
                "{}",
                "═════════════════════════════════════════".bright_cyan()
            );
            eprintln!("{} Upload failed: {}", "✗".red().bold(), e);
            eprintln!();
            eprintln!("{}", "Troubleshooting:".bright_white().bold());
            eprintln!("  - Verify the server ID is correct");
            eprintln!("  - Check your pmcp.run authentication: cargo pmcp auth login");
            eprintln!(
                "  - Ensure the server exists: https://pmcp.run/servers/{}",
                server_id
            );
            eprintln!(
                "{}",
                "═════════════════════════════════════════".bright_cyan()
            );
            anyhow::bail!("Upload failed: {}", e);
        },
    }

    Ok(())
}
```

Key design decisions:
1. **Single file path** (not Vec<PathBuf> like test upload) — loadtest configs are single files
2. **Validate BEFORE auth** — fail fast on bad configs without wasting time on OAuth flow
3. **LoadTestConfig::from_toml()** handles both TOML parsing AND semantic validation (scenarios exist, weights > 0)
4. **Actionable error messages** — tell the user exactly what's wrong and how to fix it
5. **Next-steps guidance** — dashboard link, cloud trigger hint, local run alternative
6. **Config ID + version** displayed on success — so user can reference the upload
