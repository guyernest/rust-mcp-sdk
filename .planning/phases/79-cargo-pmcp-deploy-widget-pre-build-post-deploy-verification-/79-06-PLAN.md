<objective>
Close the single open gap from `79-UAT.md` Test 3 (severity: major, status: diagnosed) — `cargo pmcp deploy` widget pre-build hard-crashes with raw `os error 2` ("No such file or directory") when a `widgets/` directory contains raw HTML / CDN-import widgets (e.g. `keypad.html` importing from `https://esm.sh/...`) and has no `package.json`. Worse, before crashing the CLI invokes `npm install` from the widgets dir; npm walks UP the directory tree, audits 1839 packages from a parent workspace, and risks writing outside the project (`node_modules/` or `package-lock.json` mutation in a parent dir).

This is the documented zero-build "Raw HTML" MCP Apps use case from Phase 45 onwards (single-file widgets with CDN imports). The bug regresses that contract. Phase 79 Wave 2 (`run_widget_build`) was written under the assumption that any auto-detected `widget/` or `widgets/` dir is a Node project, which is false for the raw-HTML archetype.

**Root cause** (already located by `gsd-debugger`, recorded in `79-UAT.md` Gaps section):

- `cargo-pmcp/src/deployment/widgets.rs:269-289` — `detect_widgets()` synthesises a Node-shaped `WidgetConfig` for any auto-detected `widget/` or `widgets/` dir without checking for `package.json`.
- `cargo-pmcp/src/deployment/widgets.rs:362-374` — `run_widget_build()` enters the Node pipeline (`ensure_node_modules` → `invoke_build_script`) unconditionally.
- `cargo-pmcp/src/deployment/widgets.rs:380-404` — `ensure_node_modules()` only short-circuits on `node_modules/` present OR Yarn-PnP markers; it does NOT check for `package.json`. So it spawns `npm install` against a directory with no manifest, causing npm to walk up the tree.
- `cargo-pmcp/src/deployment/widgets.rs:493-507` — `verify_build_script_exists()` calls `fs::read_to_string(pkg_json_path)` first, raising raw `io::Error` when the file is missing; the friendly `bail!` at line 502 (no `build` script) is unreachable when the file itself doesn't exist.

**Bug surface:** Path A (deploy pre-build) AFFECTED. Path B (`cargo pmcp test apps`) NOT affected — `scan_widgets_dir` already walks `*.html` only and never touches `package.json` or `npm`.

**Fix shape:** Add a single `is_node_project(widget_dir) -> bool` helper, branch on it at the TOP of `run_widget_build` (early-return `Ok(resolved)` on the raw-HTML path so the orchestrator still gets `ResolvedPaths` and `PMCP_WIDGET_DIRS` is populated correctly per HIGH-C1), and add a defense-in-depth `Path::is_file()` guard at the head of `verify_build_script_exists` that bails with a friendly diagnostic BEFORE `read_to_string`. Pure additive guard inside `cargo-pmcp` — no schema change, no API break, no downstream supplements.

Purpose: Restore the documented zero-config raw-HTML widget contract; eliminate the npm parent-walk security/correctness risk; replace the raw OS error with an actionable diagnostic; preserve all REVISION 3 invariants (HIGH-G2 rollback message, HIGH-C1 `PMCP_WIDGET_DIRS` colon-join, HIGH-G1 build.rs local-discovery).

```rust
fn verify_build_script_exists(widget_dir: &Path) -> AnyhowResult<()> {
    let pkg_json_path = widget_dir.join("package.json");

    // Defense-in-depth (UAT Test 3 supersession): bail with a friendly
    // diagnostic BEFORE std::fs::read_to_string raises raw io::Error
    // ("os error 2"). The early-return in run_widget_build covers the
    // common path; this guard covers the "explicit Node-shaped build/install
    // argv against a non-Node dir" edge case where the orchestrator skipped
    // the early-return because explicit argv was provided.
    if !pkg_json_path.is_file() {
        bail!(
            "widget dir {} has no package.json — add one with a 'build' script, \
             configure widgets in .pmcp/deploy.toml, or remove the build = ... \
             override (raw-HTML widgets need no build).",
            widget_dir.display()
        );
    }

    let raw = std::fs::read_to_string(&pkg_json_path)
        .with_context(|| format!("Failed to read {}", pkg_json_path.display()))?;
    let parsed: serde_json::Value = serde_json::from_str(&raw)
        .with_context(|| format!("Invalid JSON in {}", pkg_json_path.display()))?;
    let has_build = parsed.get("scripts").and_then(|s| s.get("build")).is_some();
    if !has_build {
        bail!(
            "package.json at {} has no 'build' script — add one or configure widgets in .pmcp/deploy.toml",
            pkg_json_path.display()
        );
    }
    Ok(())
}
```
