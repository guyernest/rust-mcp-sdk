<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → streamable-HTTP server | untrusted JSON-RPC over the network; the SDK adapter applies CORS/DNS-rebind/header-validation layers |
| --schema file content → public MCP resource | admin-redacted file becomes a client-readable resource (D-05), merged into the configured schema resource |
| configured resources/prompts → client surface | the full configured resource + prompt set is exposed; only the schema resource's content is overridden |
| non-SQLite connector construction → no live backend | lazy startup must not require or leak creds; timeout-guarded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-05-01 | Spoofing | local HTTP transport | mitigate | StreamableHttpServer::start applies the DnsRebindingLayer (origin-locked, AllowedOrigins::localhost() in default config) + SecurityHeaders — reuse the Phase 56 adapter, do NOT hand-roll axum |
| T-85-05-02 | Information disclosure | merged schema resource | mitigate | Only the schema resource's content is replaced with the admin-redacted --schema file (D-05); no live schema_text() introspection path; the other configured resources pass through unchanged (REVIEW FIX #2) |
| T-85-05-03 | Information disclosure | non-SQLite lazy startup | mitigate | Dispatch uses connect_lazy/from_config + offline athena (Phase 84 D-09 + Plan 04 review fix); SC-1 test runs with NO creds env, timeout-guarded so an accidental network call FAILS rather than hangs |
| T-85-05-04 | Tampering | request body size | accept | StreamableHttpServerConfig carries max_request_bytes (DEFAULT_MAX_REQUEST_BYTES) — SDK-owned bound, accepted as-is for the local binary |
</threat_model>
