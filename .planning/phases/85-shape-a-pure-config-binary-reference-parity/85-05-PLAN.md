<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → streamable-HTTP server | untrusted JSON-RPC over the network; the SDK adapter applies CORS/DNS-rebind/header-validation layers |
| --schema file content → public MCP resource | admin-redacted file becomes a client-readable resource (D-05) |
| non-SQLite connector construction → no live backend | lazy startup must not require or leak creds |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-04-01 | Spoofing | local HTTP transport | mitigate | StreamableHttpServer::start applies the DnsRebindingLayer (origin-locked, AllowedOrigins::localhost() in default config) + SecurityHeaders — reuse the Phase 56 adapter, do NOT hand-roll axum |
| T-85-04-02 | Information disclosure | schema resource | mitigate | Resource content is the admin-redacted --schema file verbatim (D-05); no live schema_text() introspection path in assemble — what's in the file is what's exposed |
| T-85-04-03 | Information disclosure | non-SQLite lazy startup | mitigate | Dispatch uses connect_lazy/from_config (Phase 84 D-09) — no creds required to build the connector or serve tools/list; the SC-1 test runs with NO creds env and must not hang or error |
| T-85-04-04 | Tampering | request body size | accept | StreamableHttpServerConfig carries max_request_bytes (DEFAULT_MAX_REQUEST_BYTES) — SDK-owned bound, accepted as-is for the local binary |
</threat_model>
