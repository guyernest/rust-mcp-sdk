<objective>
Build the two binary-side seams the server assembly needs: the clap CLI surface (`--config`/`--schema`/`--http`) and the NOVEL `[database] type` → `Arc<dyn SqlConnector>` dispatch (RESEARCH/PATTERNS flag dispatch as no-analog). A `type` whose feature was compiled out must produce a clear, actionable error naming the missing feature (D-08), and dispatch errors must never leak connection credentials (V7).

Purpose: Dispatch is the one new business seam in the binary; everything else is wiring of toolkit primitives. The connector trait object it returns feeds the toolkit builder chain in Plan 05. The CLI honors the D-03 two-input model: `--config` is the server+operation+code-mode-policy TOML, `--schema` is the (DDL) code-mode schema resource file.

**REVIEW FIX (Codex/Gemini MEDIUM/LOW — Athena lazy-startup must be truly offline).** D-09 requires connector construction WITHOUT a live backend or creds, and SC-1 (Plan 05) tests `tools/list` for the Athena config with NO creds. `AthenaConnector::from_config` builds an SDK client — confirm at execution time that this does NOT trigger the AWS provider chain / IMDS / network at construction time. If the constructor DOES touch the provider chain, the Athena dispatch arm MUST pass an explicit offline/static credential + region config (or use a dev/no-network constructor) so the SC-1 lazy test cannot hang or reach the network. Plan 05's SC-1 test additionally wraps non-SQLite startup in a timeout guard; this plan's job is to make the Athena arm offline-safe at the source.
Output: cli.rs, dispatch.rs, and a dispatch unit/property test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args (--config/--schema/--http) → process | admin-controlled paths; missing/invalid args fail fast |
| config [database] url/credentials → DispatchError | error messages must not leak connection secrets |
| athena dispatch → AWS provider chain | construction must stay offline; no IMDS/network at dispatch time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-04-01 | Information disclosure | DispatchError::Display | mitigate | Errors name backend/feature only; never echo url/file_path/credential substrings (property test asserts no leak) — V7 + T-84-04-02 precedent |
| T-85-04-02 | Denial of service | clap parse of malformed args | accept | clap derive handles arg validation; missing required --config/--schema fails fast with a usage error |
| T-85-04-03 | Tampering | unknown/compiled-out backend type | mitigate | unknown_backend / feature_missing errors are explicit and actionable; no silent fallback to a wrong backend |
| T-85-04-04 | Denial of service | athena construction touching AWS provider chain | mitigate | Athena arm constructs offline (explicit region, no load_from_env provider probe) so the SC-1 no-creds test cannot hang/reach the network (REVIEW FIX; Plan 05 adds a timeout guard) |
</threat_model>
