<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → execute_code | `variables` is untrusted client input; binding (not string-interpolation) preserves parameterized-query safety |
| process supervisor → binary exit code | A serve-task panic must surface as non-zero so a supervisor restarts it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-10-01 | Tampering | execute_code variables silent drop | mitigate | A schema-advertised input that is silently dropped invites clients to string-interpolate variables into `code`, bypassing parameter binding. Binding them (or explicitly rejecting) keeps the parameterized-binding safety the channel implies (WR-02). |
| T-85-10-02 | Denial of Service | serve-task panic exits 0 | mitigate | A discarded JoinError hides a crashed listener (process looks healthy, serves nothing). Propagating it as RunError::Serving → non-zero exit lets a supervisor restart. |
| T-85-10-03 | Spoofing / weak secret | empty token_secret env | mitigate | An empty env value previously flowed to the HMAC layer as a (too-short) secret; now a clear "set but empty" error stops startup with a real key, never a degenerate one. |
| T-85-10-04 | Information Disclosure | sqlite database fallback error | accept | The MissingField error names the config FIELDS, never a path; SqliteOpen stays path-free (carried from T-85-04-01). |
</threat_model>
