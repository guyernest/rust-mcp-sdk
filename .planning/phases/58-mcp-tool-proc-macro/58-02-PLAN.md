<objective>
Implement the `#[mcp_server]` impl-block companion macro that collects `#[mcp_tool]` methods, generates per-tool ToolHandler structs using `&self` access, and provides bulk registration via `register_tools()` or `.mcp_server()` builder extension.

Purpose: The impl-block pattern is the "killer feature" (per team feedback) for servers with 3+ tools. It eliminates `N x TypedToolWithOutput::new(...)` boilerplate, gives natural `&self` state access (D-09), and supports generic impl blocks for composition servers (D-25). This plan completes the macro pair: standalone functions (Plan 01) + impl blocks (this plan).

Output: mcp_server.rs with impl-block macro expansion, #[mcp_server] entry point, and McpServer builder extension trait.

**Create `pmcp-macros/src/mcp_server.rs`** implementing the `#[mcp_server]` attribute macro for impl blocks.

**High-level flow:**

The macro processes an `ItemImpl`, finds all methods with `#[mcp_tool(...)]` attributes, and for each:
1. Generates a private handler struct `{PascalName}ToolHandler` that holds `Arc<ServerType>` and implements `ToolHandler`
2. Generates a `register_tools(self, builder: pmcp::ServerBuilder) -> pmcp::ServerBuilder` method on the server struct
3. Preserves the original impl block with all methods (stripping `#[mcp_tool]` attributes from methods)

**Implementation details:**

A. **Collect tool methods:**
   - Iterate `input.items`, find `ImplItem::Fn` with `#[mcp_tool(...)]` attribute
   - Parse each `#[mcp_tool(...)]` attribute using the same `McpToolArgs` struct from mcp_tool.rs (make its fields and struct `pub(crate)`)
   - For each method, extract: method name, tool_name (from args.name or method name per D-06), description, is_async, parameter roles (skip `&self` via classify_param), return type

B. **Generate per-tool handler structs:**

   For a method `query(&self, args: QueryArgs) -> Result<QueryResult>` on `MyServer`:

   ```rust
   struct QueryToolHandler {
       server: std::sync::Arc<MyServer>,
   }

   #[async_trait::async_trait]
   impl pmcp::ToolHandler for QueryToolHandler {
       async fn handle(&self, args: serde_json::Value, _extra: pmcp::RequestHandlerExtra) -> pmcp::Result<serde_json::Value> {
           let typed_args: QueryArgs = serde_json::from_value(args)
               .map_err(|e| pmcp::Error::invalid_params(format!("Invalid arguments for tool 'query': {}", e)))?;
           let result = self.server.query(typed_args).await?;
           serde_json::to_value(result).map_err(|e| pmcp::Error::internal(format!("Failed to serialize result: {}", e)))
       }

       fn metadata(&self) -> Option<pmcp::types::ToolInfo> {
           // Use same branching ToolInfo constructor pattern as mcp_tool.rs:
           // - If annotations present: ToolInfo::with_annotations(...)
           // - If no annotations: ToolInfo::new(...)
           // - Chain .with_output_schema() if typed output
       }
   }
   ```

   For async methods: `self.server.method_name(args).await?`
   For sync methods: `self.server.method_name(args)?`
   If method takes `extra: RequestHandlerExtra`: pass it through. If not: use `_extra` in generated handle() signature.

C. **Handle generic impl blocks (D-25):**

   For `impl<F: FoundationClient + 'static> ArithmeticsServer<F>`:
   - Extract generics from `input.generics`
   - Add Send + Sync + 'static bounds via `utils::add_async_trait_bounds` for generated handler structs
   - Propagate generics to handler struct: `struct CalculateToolHandler<F: FoundationClient + Send + Sync + 'static> { server: Arc<ArithmeticsServer<F>> }`
   - Use the same generics on the ToolHandler impl
   - Apply to the `register_tools()` impl block as well

D. **Generate `register_tools()` method:**

   Add a new impl block (separate from the user's impl block) on the server type:

   ```rust
   impl MyServer {
       /// Register all #[mcp_tool] methods as tools on the builder.
       pub fn register_tools(self, mut builder: pmcp::ServerBuilder) -> pmcp::ServerBuilder {
           let shared = std::sync::Arc::new(self);
           builder = builder.tool("query", QueryToolHandler { server: shared.clone() });
           builder = builder.tool("clear_cache", ClearCacheToolHandler { server: shared.clone() });
           // ... for each tool method
           builder
       }
   }
   ```

   For generic servers: the impl block and handler structs carry the same generics.

E. **Strip `#[mcp_tool]` attributes from methods:**

   After collecting tool info, remove the `#[mcp_tool(...)]` attribute from each method in the original impl block so the method compiles as a normal method. The macro output includes:
   - The original impl block (with #[mcp_tool] attributes stripped)
   - Per-tool handler structs and ToolHandler impls
   - The register_tools() impl block

F. **Error handling:**
   - No `#[mcp_tool]` methods found: emit compile error "No methods marked with #[mcp_tool] found in impl block"
   - Method has `State<T>` parameter: emit compile error "#[mcp_server] methods use &self for state access, not State<T>"

**Add `#[mcp_server]` entry point in `pmcp-macros/src/lib.rs`:**

After the `#[mcp_tool]` entry point:

```rust
/// Collects `#[mcp_tool]` methods from an impl block and generates tool handlers.
///
/// # Examples
///
/// ```rust,ignore
/// #[mcp_server]
/// impl MyServer {
///     #[mcp_tool(description = "Query database")]
///     async fn query(&self, args: QueryArgs) -> Result<QueryResult> {
///         self.db.execute(&args.sql).await
///     }
/// }
///
/// // Register all tools at once:
/// let builder = my_server.register_tools(builder);
/// ```
#[proc_macro_attribute]
pub fn mcp_server(args: TokenStream, input: TokenStream) -> TokenStream {
    let input = parse_macro_input!(input as ItemImpl);
    mcp_server::expand_mcp_server(args.into(), input)
        .unwrap_or_else(|err| err.to_compile_error())
        .into()
}
```

/// Register all tools from an `#[mcp_server]` annotated type.
///
/// This is the ergonomic counterpart to individually registering tools.
/// The server instance provides shared state via `&self` to all tool methods.
///
/// # Examples
///
/// ```rust,ignore
/// use pmcp::ServerBuilder;
///
/// #[mcp_server]
/// impl MyServer {
///     #[mcp_tool(description = "Query data")]
///     async fn query(&self, args: QueryArgs) -> Result<Value> { /* ... */ }
/// }
///
/// let server = MyServer { db };
/// let builder = ServerBuilder::new()
///     .name("my-server")
///     .mcp_server(server);
/// ```
pub fn mcp_server<T: McpServer>(self, server: T) -> Self {
    server.register_tools(self)
}
```

/// Trait for types annotated with `#[mcp_server]`.
