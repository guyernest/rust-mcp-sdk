## Deviations from Plan

### Bundled commits (planner-permitted)

**1. [Planner-permitted] Task 2 delivered as one atomic commit (4 files)**
- **Found during:** Task 2 execution
- **What:** PLAN Task 2 lists 4 consumer files and the `<action>` explicitly directs "Rewrite all four consumer files in a single atomic commit so the template type-checks at every step."
- **Outcome:** Commit `a488184f` contains all four rewrites simultaneously.
- **Why:** Avoids a type-error window between commits (e.g., rewriting signup/page.tsx to import the hook before Header.tsx stops reading NEXT_PUBLIC_* would temporarily break the TS project check). The planner pre-authorized this bundling.
- **Not a Rule-1/2/3 deviation** — this was executed per PLAN.md's explicit direction.

### Code-level deviations from CR-03 verbatim snippets

**1. [Rule 1 — Clarity/lint-safety] signup/page.tsx `disabled` expression explicit-null guard**
- **Found during:** Task 2 (`app/signup/page.tsx` rewrite)
- **Issue:** CR-03 §2's verbatim snippet uses `const disabled = cfg === 'error' || (cfg && !cfg.signup)` — but the string `'error'` is truthy, so `cfg && !cfg.signup` would evaluate when `cfg === 'error'` and the implicit-truthiness pattern flags on strict lints.
- **Fix:** Used `const disabled = cfg === 'error' || (cfg !== null && !cfg.signup)` — explicit null guard, behavior identical to CR-03 intent.
- **Files modified:** cargo-pmcp/templates/landing/nextjs/app/signup/page.tsx
- **Verification:** `tsc --noEmit` green; AC-5 manual check passes (disabled message renders on /landing-config block).
- **Committed in:** a488184f (Task 2 commit)
- **Documented in PLAN:** PLAN lines 377-381 note this deviation up front ("Deviations from CR-03 §2: added explicit `cfg !== null` in the `disabled` expression to avoid the implicit-truthiness lint firing on `cfg && !cfg.signup` when `cfg === 'error'` (the string `'error'` is truthy).").

**Total deviations:** 1 minor code-level deviation (lint-safety); the Task 2 four-file bundling was planner-authorized and is not a deviation.
**Impact on plan:** Zero — deviation documented up-front in PLAN and matches CR-03 §2 behavior intent exactly. No scope creep. No new dependencies. No breaking changes.
