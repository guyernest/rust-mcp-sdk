### G5 specifically — why tightened (Codex L3)

The earlier G5 form was `grep -c 'cfg.mcpEndpoint\|endpoint' ≥ 4`, which only proved the file mentioned the word `endpoint` enough times. That could be satisfied by a file that happened to have four occurrences of the local `const endpoint` alias but never actually used `cfg.mcpEndpoint` — or worse, used a mixture where some snippets still used a hand-assembled URL.

The tightened form asserts two things separately:
1. **Positive:** `grep -c 'cfg.mcpEndpoint' ≥ 4` — the literal `cfg.mcpEndpoint` (possibly via a `const endpoint = cfg.mcpEndpoint` declaration + 3 `${endpoint}` uses, or 4 direct inlines) must appear at least four times in `connect/page.tsx`.
2. **Negative:** `! grep -E '\.us-east\.true-mcp\.com|\$\{serverName\}'` — no legacy hand-assembly style (domain substring or `${serverName}` interpolation) remains.

Together these assert every MCP endpoint reference flows through `cfg.mcpEndpoint` and no leftover hand-assembly exists. Matches AC-8's intent.

### G7 specifically — why added (Codex M3)

The original plan assumed "template source was changed" implies "`pmcp new --template landing` scaffolds correctly", but Task 5 never exercised the scaffold command. If the scaffold pipeline has an explicit file manifest / embed / index step (e.g., if a later change adds `include_bytes!` per template file or an `index.json`-style manifest), a newly-added file like `lib/useLandingConfig.ts` could be silently dropped from the scaffold output.

G7 runs the actual command against a temp dir and asserts the scaffolded project contains the hook and all four consumers import it. The command signature (`cargo pmcp landing init --template nextjs --output <dir> --server-name <name>`) is confirmed against `cargo-pmcp/src/commands/landing/init.rs::init_landing_page(project_root, template_name, output, server_name)`.

Note: CR-03 AC-1 says `pmcp new --template landing`, but this repo has no `pmcp new` subcommand — AC-1's phrasing is loose platform-team wording. G7 uses the real command.
