## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed golden bytes → BundleLoader | The golden is the test oracle; it must pass the same integrity gate real bundles do |
| generator non-determinism → byte-stability | Map ordering / float formatting could make regeneration non-reproducible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-92-05 | Tampering | golden fixture drift vs runtime hasher | mitigate | The golden BUNDLE.lock is produced by the runtime build_bundle_lock and `golden_passes_boot_integrity` asserts the loader accepts it (Task 2) |
| T-92-06 | Repudiation | non-deterministic regeneration | mitigate | `golden_regeneration_is_byte_identical` is a CI check; the generator uses a fixed serde config and no unordered iteration (Tasks 1-2) |
| T-92-07 | Information Disclosure | customer data leak into the fixture | mitigate | D-01 hard constraint; grep-gate `ufh|quote|coil|towelrad` = 0 in fixtures + support (Tasks 1-3) |
| T-92-08 | Tampering | negative-path test using a committed corrupt fixture | accept→mitigate | D-05: tamper-at-test-time only; one golden committed, corruption happens in a tempdir copy (Task 3) |
</threat_model>
