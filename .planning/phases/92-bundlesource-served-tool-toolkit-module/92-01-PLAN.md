<objective>
Build the runtime-side bundle contract: a dumb-byte `BundleSource` trait
(local-dir + embedded impls) and a single shared `BundleLoader` that performs
parse + fail-closed integrity verification identically for every source, plus
the additive runtime model changes this phase requires (D-17 `workflow`→
`bundle_id` rename, drop the `supply_total_cell` headline field, add the D-18
`Manifest.annotations` field).

Purpose: Freeze the consumer side of the bundle contract from the leaf crate.
Because `BundleSource` returns only raw bytes and `BundleLoader` is the one
verifier, no source impl can skip WBSV-08. The runtime stays reader-free.
Output: `bundle_source.rs`, `bundle_loader.rs`, edited `manifest_model.rs`/
`artifact_model.rs`/`lib.rs`/`Cargo.toml` in `pmcp-workbook-runtime`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| on-disk/embedded bundle bytes → BundleLoader | A bundle artifact may be tampered, swapped, or truncated before boot |
| supply chain → new include_dir dep | A malicious/typo-squatted embedding crate enters the runtime tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-92-01 | Tampering | bundle artifact (single-file swap/byte-flip) | mitigate | BundleLoader recomputes per-artifact + combined BUNDLE.lock hash via the runtime's own build_bundle_lock and returns IntegrityMismatch fail-closed (Task 3) |
| T-92-02 | Spoofing | BUNDLE.lock provenance/identity triple desync | mitigate | verify_stamp_binding cross-checks the lock identity against hash-covered members; returns StampMismatch (Task 3) |
| T-92-03 | Tampering | BundleSource impl returning a pre-parsed bundle (WBSV-08 bypass) | mitigate | Trait is a dumb byte accessor only (read_artifact/list_artifacts); the single shared BundleLoader is the only parse+verify path — bypass is type-level impossible (Task 2) |
| T-92-SC | Tampering | include_dir cargo install | mitigate | slopcheck unavailable → [ASSUMED]; blocking human-verify checkpoint (Task 1) confirms author/license/repo + cargo audit before install; never auto-approvable |
| T-92-04 | Denial of Service | malformed/oversized bundle member | mitigate | Total, panic-free parse returns BundleLoadError::Parse; crate-level #![deny(panic)] enforces panic-freedom (Task 3) |
</threat_model>
