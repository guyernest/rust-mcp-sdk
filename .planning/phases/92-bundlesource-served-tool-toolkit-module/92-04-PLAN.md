<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client-supplied workbook:// URI → resources/read | The pointer round-trips through the client; the URI is attacker-controlled |
| decoded inputs → executor | Decoded inputs must be re-validated before re-run (injection lever) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-92-14 | Denial of Service | oversized workbook:// URI | mitigate | MAX_ENCODED_URI_LEN size guard checked FIRST, before any decode (Task 1) |
| T-92-15 | Spoofing | cross-provenance / forged workbook:// URI | mitigate | render_resource verifies decoded provenance == bundle stamp before rendering (Task 2) |
| T-92-16 | Tampering | injected inputs via the round-tripped URI | mitigate | RE-VALIDATE decoded inputs through validate_input on every read before re-run (Task 2) |
| T-92-17 | Denial of Service | malformed URI panic | mitigate | Total, panic-free decode; #![deny(panic)] enforces it; a proptest decode-fuzz test (`prop_decode_total`) proves totality over arbitrary/adversarial input (Task 1) |
| T-92-18 | Information Disclosure | server-side render state leakage | accept | Stateless regen-on-read: no session, bytes recomputed per read (V3) — no server-side render state to leak (Task 2) |
</threat_model>
