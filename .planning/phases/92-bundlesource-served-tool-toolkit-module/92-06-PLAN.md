<objective>
Close Blocker 1 (CR-01) from 92-VERIFICATION.md: caller-supplied inputs are
silently ignored — every calculate / explain / render_workbook call computes
from the bundle's baked-in default inputs (gross_income=60000) regardless of
what the caller sends. The root cause is that the fixture generator emits the
three Role::Input cells as IR literals, which the executor's literal arm
unconditionally re-seeds at topo-walk time, overwriting validate_input's
caller-seeded values.

This plan applies the three required fixes plus the recommended defense-in-depth:
1. build_ir() omits Role::Input cells from the IR (honoring the executor's
   documented seed contract).
2. The executor literal arm becomes seed-preserving (a future compiler-emitted
   bundle that repeats this shape cannot reintroduce the bug silently).
3. A value-asserting regression test proves a non-default input flows through.
4. The committed golden is regenerated so it is byte-identical to the fixed
   generator (the always-on byte-stability test enforces this in CI).

Purpose: WBSV-01 (calculate returns all named outputs from the caller's inputs),
WBSV-02 (explain trace reflects the caller's inputs), WBSV-05 (render_workbook
regen-on-read reflects the caller's inputs) are all BLOCKED by this one defect.
The architecture is sound; the committed golden bundle violates the
architecture's own contract.

Output: a fixed generator, a seed-preserving executor, a regenerated golden, and
a regression test that would have caught this.
</objective>
