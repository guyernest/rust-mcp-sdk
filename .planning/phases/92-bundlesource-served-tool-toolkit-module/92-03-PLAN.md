## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent tool input → calculate/explain | Untrusted JSON crosses into the executor; must be validated fail-closed |
| manifest skew → enum/role gates | A skewed manifest could fail-open a numeric-enum or role-less input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-92-09 | Tampering | malicious/adversarial tool input (out-of-tier/dtype/enum/oversized) | mitigate | validate_input fail-closed: WR-05 no-role rejection, WR-02 string-only enum gate, V4 strict-constant rejection; a proptest fuzz test proves validate_input is total (never panics) over arbitrary inputs (Task 2) |
| T-92-10 | Tampering | domain failure masquerading as success | mitigate | to_iserror_result returns isError:true in structuredContent; pmcp tool dispatch hardcodes is_error:false so structuredContent is the only machine-actionable path (Tasks 1,3) |
| T-92-11 | Tampering | NaN/Infinity output → JSON null masquerading as success | mitigate | Finiteness check (WR-06) on every numeric output; non-finite → error (Task 3) |
| T-92-12 | Spoofing | response not bound to a verified bundle | mitigate | Every response + error envelope carries the ProvStamp (bundle_id+version+combined hash) computed by the boot loader (Tasks 1,3) |
| T-92-13 | Information Disclosure | customer logic re-introduced during the lift | mitigate | S-1/S-2/S-4 scrub deltas with grep-gates on every workbook/*.rs file (all tasks) |
</threat_model>
