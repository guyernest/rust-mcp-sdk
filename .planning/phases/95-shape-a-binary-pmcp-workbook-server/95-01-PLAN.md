| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-01 | Tampering | bundle dir → `build_server` boot gate | mitigate | `try_with_workbook_bundle` / `load_bundle` recompute BUNDLE.lock hashes BEFORE registering any tool; failure → `RunError::Bundle` → non-zero exit. The binary adds no served logic and never serves a partial/tampered bundle. The assembly-time `try_with_workbook_bundle` load is the integrity boundary: it INDEPENDENTLY recomputes the lock hashes even when an earlier `--bundle-id` pre-load already ran (see T-95-02). |
| T-95-02 | Spoofing / TOCTOU | wrong-bundle served silently; swap window between id check and assembly | mitigate | `--bundle-id` assertion (D-01): pre-load via `load_bundle`, compare loaded `bundle.stamp.bundle_id` to the operator-typed id, return `RunError::BundleIdMismatch` (fail-closed) BEFORE assembly. The bundle is read TWICE (id-check load + assembly load — the toolkit `try_with_workbook_bundle` re-loads internally, VERIFIED workbook/mod.rs:230, and exposes no preloaded-bundle entrypoint). This is NOT an exploitable TOCTOU: the SECOND (assembly) load re-verifies BUNDLE.lock hashes fail-closed, so a bundle swapped between the two loads is rejected at assembly unless the swap preserves every lock hash (i.e. is the same verified bundle). Single-load would require a toolkit API change (Phase 92 scope) — deliberately deferred; the accepted double-read is documented in assemble.rs and covered by the mismatch + tampered-bundle unit tests. |
| T-95-03 | Denial of Service | crashed serve task looks healthy | mitigate | `handle.await.map_err(RunError::Serving)?` propagates the serve-task `JoinError` (T-85-10-02), so a crashed listener exits non-zero rather than appearing alive. |
| T-95-04 | Information Disclosure | error `Display` leaking paths/secrets | mitigate | `BundleIdMismatch` echoes only operator-typed `bundle_id` strings (credential-free, mirrors `DispatchError`); raw `--bundle-dir` filesystem paths are not echoed into MCP-client-visible errors (tracing-only at the binary's discretion). |
| T-95-05 | Elevation/Exposure | public listener out of the box; bad bind address | accept (mitigated by default) | `--http` defaults to `127.0.0.1:8080` loopback (D-04); an invalid `--http` value fails CLOSED at `run_serving` (parsed to `SocketAddr`; failure → `RunError::Addr` → non-zero exit, covered by the invalid-http unit test) rather than binding an unintended address. DNS-rebinding/CORS/security-header layers come from the SDK's Tower/axum `StreamableHttpServer` adapter and are NOT hand-rolled. Operator opting into a public bind is an explicit choice. |
| T-95-SC | Tampering | npm/pip/cargo installs | n/a | No new package-manager installs — all deps are existing in-workspace path crates already audited in prior phases. No legitimacy checkpoint required. |
