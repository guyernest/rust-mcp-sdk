<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → binary (CLI args) | `--bundle-dir` / `--bundle-id` / `--http` are operator-supplied; `--bundle-dir` points at on-disk bundle artifacts that are integrity-verified before serving |
| bundle dir → served tools | the on-disk `bundle@version` directory is the data source; tampered/incomplete bundles must never serve (fail-closed boot gate) |
| MCP client → HTTP listener | remote callers hit the streamable-HTTP listener; default bind is loopback (D-04) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-01 | Tampering | bundle dir → `build_server` boot gate | mitigate | `try_with_workbook_bundle` / `load_bundle` recompute BUNDLE.lock hashes BEFORE registering any tool; failure → `RunError::Bundle` → non-zero exit. The binary adds no served logic and never serves a partial/tampered bundle. |
| T-95-02 | Spoofing | wrong-bundle served silently | mitigate | `--bundle-id` assertion (D-01): pre-load, compare loaded `bundle.stamp.bundle_id` to the operator-typed id, return `RunError::BundleIdMismatch` (fail-closed) before assembly. |
| T-95-03 | Denial of Service | crashed serve task looks healthy | mitigate | `handle.await.map_err(RunError::Serving)?` propagates the serve-task `JoinError` (T-85-10-02), so a crashed listener exits non-zero rather than appearing alive. |
| T-95-04 | Information Disclosure | error `Display` leaking paths/secrets | mitigate | `BundleIdMismatch` echoes only operator-typed `bundle_id` strings (credential-free, mirrors `DispatchError`); raw `--bundle-dir` filesystem paths are not echoed into MCP-client-visible errors (tracing-only at the binary's discretion). |
| T-95-05 | Elevation/Exposure | public listener out of the box | accept (mitigated by default) | `--http` defaults to `127.0.0.1:8080` loopback (D-04); DNS-rebinding/CORS/security-header layers come from the SDK's Tower/axum `StreamableHttpServer` adapter and are NOT hand-rolled. Operator opting into a public bind is an explicit choice. |
| T-95-SC | Tampering | npm/pip/cargo installs | n/a | No new package-manager installs — all deps are existing in-workspace path crates already audited in prior phases. No legitimacy checkpoint required. |
</threat_model>
