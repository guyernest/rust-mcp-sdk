## Noted Deviation: src/ change to auth.rs

**Boundary stated in SPEC:** "No change to AuthConfig, the passthrough chain, the connector, or other toolkit/pmcp-openapi-server src/ code — this phase is config + fixture + docs + example/test Rust only."

**What actually shipped:** `crates/pmcp-server-toolkit/src/http/auth.rs` received two `#[serde(alias)]` attributes:
- Line 109: `#[serde(alias = "oauth2_client_credentials")]` on `OAuth2ClientCredentials`
- Line 134: `#[serde(alias = "oauth_passthrough")]` on `OAuthPassthrough`

**Assessment: JUSTIFIED — real upstream bug, minimal, fully tested, backward-compatible.**

The bug: `rename_all = "snake_case"` on the `AuthConfig` enum derives the tag `o_auth_passthrough` (not `oauth_passthrough`). The SPEC-mandated config form — `type = "oauth_passthrough"` — never deserialized at runtime. The example literally could not function without this fix.

The fix is unambiguously additive:
- `#[serde(alias)]` adds an accepted deserialization spelling WITHOUT changing the canonical serialization output
- The derived `o_auth_passthrough` canonical form still round-trips, proven by `test_snake_case_tag_still_deserializes_after_alias`
- Three targeted unit tests were added (WR-01 fix): `test_oauth_passthrough_documented_tag_deserializes`, `test_oauth2_client_credentials_documented_tag_deserializes`, `test_snake_case_tag_still_deserializes_after_alias`
- The auth regression suite passes 27/27, including all three new alias tests
- Only `crates/pmcp-server-toolkit/src/http/auth.rs` was modified; `crates/pmcp-openapi-server/src/` was not touched

Review findings WR-02 (empty ordersAddr guard for C006) and WR-03 (course chapter step misnumbering) were both resolved before verification:
- WR-02: `if (ordersAddr === "") { return []; }` guard present in both TOML files (line 116 fixture, line 122 examples)
- WR-03: Step numbers dropped entirely from course chapter; descriptive headers used instead
