## (a) Canonical workbook json — path, dates, addressing convention

- **Path:** `crates/pmcp-openapi-server/tests/fixtures/contoso-m365-workbook.json`
- **`ref_date`:** `"2026-05-28"` (Plan 03's `const REF` copies this verbatim)
- **`window_start`:** `"2026-02-28"` = `ref_date` minus 3 calendar months, written as a literal. `ref_date` day is 28 (<= 28) so the boundary is calendar-unambiguous. The headline window is the inclusive interval `[2026-02-28, 2026-05-28]`.
- **Drive/item id placeholders:** `drive_id="CONTOSO_DRIVE"`, `customers_item_id="CUSTOMERS_ITEM"`, `orders_item_id="ORDERS_ITEM"`.
- **Customers addressing:** Customers sheet row 1 = header, data from row 2. `customer_id` `C00N` -> sheet row `N + 1` -> single-customer range `A{row}:D{row}` over columns `customer_id,name,segment,region`. `all_customers_address = "A2:D7"`.
- **Orders addressing:** Orders sheet row 1 = header, orders grouped/sorted by `customer_id` so each customer's orders form a contiguous block. Per-id block addresses in `orders_blocks`. `all_orders_address = "A2:D7"`. Columns `order_id,customer_id,order_date,amount`.

- C006 = 0 — no orders at all -> NON-MATCH

C006 has no contiguous Orders block (intentional; `get_customer_orders('C006')` reads an empty result).

## (d) OData path form decision + the parse result that drove it

- **Shipped form: PRIMARY (OData parenthesized).** The spec path is
  `/drives/{drive-id}/items/{item-id}/workbook/worksheets/{worksheet-id}/range(address='{address}')`.
- **Parse result driving the decision:** a Wave-0 smoke parse through `openapiv3` 2.2.0 (same crate major `OpenApiSchema::parse` uses) accepted the parenthesized segment; then a throwaway integration test ran the REAL `pmcp_server_toolkit::http::OpenApiSchema::parse` against the shipped `contoso-m365-api.yaml` and `operation_for(<odata-path>, "GET")` resolved `getWorksheetRange` (test printed `SCRATCH_OK`, then removed before commit). **No fallback to a plain `/range` + query `address` was needed.**
- **Runtime note:** the script's runtime `range(address='...')` URL is identical either way (the spec is Code Mode CONTEXT only and is not used for dispatch — tools dispatch from the `[[tools]]` script). Plan 02 Task 1 re-asserts `OpenApiSchema::parse` accepts this form.
