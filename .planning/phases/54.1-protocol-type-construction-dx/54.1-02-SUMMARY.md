## Decisions Made
- TaskStatus Default variant is Working (most common initial state for task creation)
- Task::new() sets timestamps to empty strings rather than requiring them -- .with_timestamps() sets both at once
- ToolChoice uses static factory methods (auto/required/none) instead of ::new() since it wraps a single enum field
- CreateMessageParams has no Default derive since messages is a required field
- Pre-existing clippy warnings in core.rs and version.rs not addressed (out of scope -- not caused by this plan's changes)
