Lift the six `_arc` handler-registration methods from the private `ServerCoreBuilder` (in `src/server/builder.rs`) to the public `ServerBuilder` (in `src/server/mod.rs`), and add a `Server::get_tool` accessor symmetric with the existing `Server::get_prompt`. Purpose (per CONTEXT.md D-01, D-04, D-05, D-07, D-08): close the public-builder Arc-symmetry gap so external toolkit authors (Phase 83's `pmcp-server-toolkit` and downstream) can share `Arc<dyn Handler>` between the builder and an in-process handler map without writing a 20-line delegating wrapper shim — without touching existing `.tool()` / `.prompt()` / etc. signatures. The lift is pure additive copy (D-04) of all six `_arc` siblings (D-05). The `get_tool` accessor (D-07) gives BLDR-03's handler-level testing pattern the symmetric API surface it requires; it is folded into BLDR-03 per D-08, not a separate REQ ID.

Output: a single `src/server/mod.rs` patch adding six public `_arc` methods (one per non-Arc sibling, placed immediately after), one `get_tool` accessor (immediately after `get_prompt` at line 385), and two doctest blocks (on `get_tool` and `get_prompt`) demonstrating the testing pattern with the explicit D-03 callout.
