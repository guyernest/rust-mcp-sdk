    2. Create cargo-pmcp/src/secrets/resolve.rs with:

    ```rust
    use std::collections::HashMap;
    use std::path::Path;
    use crate::deployment::metadata::SecretRequirement;

    /// Result of resolving secrets from environment and .env files.
    #[derive(Debug, Clone)]
    pub struct SecretResolution {
        /// Resolved secrets: env_var_name -> value
        pub found: HashMap<String, String>,
        /// Requirements that could not be resolved
        pub missing: Vec<SecretRequirement>,
    }
    ```

    - `resolve_secrets(requirements: &[SecretRequirement], dotenv_vars: &HashMap<String, String>) -> SecretResolution`
      - For each requirement, compute lookup key as `req.env_var.as_deref().unwrap_or(&req.name)` (per Pitfall 4 in RESEARCH.md)
      - Check std::env::var(lookup_key) first. If Ok, insert into found. (per D-13: shell env wins)
      - Else check dotenv_vars.get(lookup_key). If Some, insert into found.
      - Else push requirement into missing.

    - `load_dotenv(project_root: &Path) -> HashMap<String, String>`
      - Build path as `project_root.join(".env")`
      - If path does not exist, return empty HashMap
      - Use `dotenvy::from_path_iter(&env_path)` to parse without mutating process env (per Pitfall 1)
      - On parse error, eprintln warning and return empty HashMap

    - `print_secret_report(resolution: &SecretResolution, server_id: &str, target: &str, global_flags: &crate::commands::GlobalFlags)`
      - Guard with `if !global_flags.should_output() { return; }`
      - If both found and missing empty: print "   No secrets required" and return
      - For found secrets: print "   Found {n} secret(s):" then each key with green checkmark (use `colored` crate's `.green()`)
      - For missing secrets on aws-lambda: print "   Missing {n} secret(s) (will warn at runtime):" then each env_name with yellow X
      - For missing secrets on pmcp-run: print per D-07 format: `"     Run: cargo pmcp secret set --server {server_id} {env_name} --target pmcp --prompt"`
      - Mark required missing secrets with "(required)" suffix

    3. In cargo-pmcp/src/secrets/mod.rs, add `pub mod resolve;` after the existing `pub mod value;` line (line 58). Add re-export: `pub use resolve::{SecretResolution, resolve_secrets, load_dotenv, print_secret_report};`

    4. Add unit tests in resolve.rs in a `#[cfg(test)] mod tests` block covering all behaviors listed above. For std::env precedence test, use `std::env::set_var`/`std::env::remove_var` with a unique key like `__PMCP_TEST_RESOLVE_KEY` to avoid collisions. Run tests with --test-threads=1 for env var safety.
