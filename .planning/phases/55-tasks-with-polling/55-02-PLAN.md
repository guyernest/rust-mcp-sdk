       a. **TaskStoreError enum:**
       ```rust
       #[derive(Debug)]
       pub enum TaskStoreError {
           NotFound { task_id: String },
           InvalidTransition { task_id: String, from: TaskStatus, to: TaskStatus },
           Expired { task_id: String },
           Internal { message: String },
       }
       ```
       Implement `std::fmt::Display` and `std::error::Error` for it. Also implement `From<TaskStoreError>` for `crate::error::Error` so it integrates with the SDK error system (map to `Error::internal(msg)`).

       b. **StoreConfig struct:**
       ```rust
       #[derive(Debug, Clone)]
       pub struct StoreConfig {
           pub default_ttl_ms: Option<u64>,      // Default: 3_600_000 (1 hour)
           pub max_ttl_ms: Option<u64>,           // Default: 86_400_000 (24 hours)
           pub default_poll_interval_ms: u64,     // Default: 5000
           pub max_tasks_per_owner: usize,        // Default: 100
       }
       ```
       Implement `Default` with the values shown above.

       c. **TaskStore trait:**
       ```rust
       #[async_trait]
       pub trait TaskStore: Send + Sync {
           async fn create(&self, owner_id: &str, ttl: Option<u64>) -> Result<Task, TaskStoreError>;
           async fn get(&self, task_id: &str, owner_id: &str) -> Result<Task, TaskStoreError>;
           async fn update_status(&self, task_id: &str, owner_id: &str, status: TaskStatus, message: Option<String>) -> Result<Task, TaskStoreError>;
           async fn list(&self, owner_id: &str, cursor: Option<&str>) -> Result<(Vec<Task>, Option<String>), TaskStoreError>;
           async fn cancel(&self, task_id: &str, owner_id: &str) -> Result<Task, TaskStoreError>;
           async fn cleanup_expired(&self) -> Result<usize, TaskStoreError>;
           fn config(&self) -> &StoreConfig;
       }
       ```
       Note: Simplified from pmcp-tasks -- no variables, no set_result, no complete_with_result. Those are PMCP extensions that stay in pmcp-tasks. The SDK trait covers the core MCP spec operations.

       d. **Internal TaskRecord struct** (not public, used by InMemoryTaskStore):
       ```rust
       struct TaskRecord {
           task: Task,
           owner_id: String,
           expires_at: Option<std::time::Instant>,
       }
       ```
       Using `std::time::Instant` for in-memory expiration (no chrono dependency needed).

       e. **InMemoryTaskStore:**
       ```rust
       pub struct InMemoryTaskStore {
           records: DashMap<String, TaskRecord>,
           config: StoreConfig,
       }
       ```
       - `new()` creates with `StoreConfig::default()`
       - `with_config(config: StoreConfig) -> Self` builder method
       - `create()`: Generate UUID v4 task_id, create Task with Working status, timestamps as ISO 8601 (use `chrono::Utc::now().to_rfc3339()`), apply default TTL if None, store in DashMap
       - `get()`: Look up by task_id, verify owner matches (return NotFound on mismatch), check expiration (return Expired if past TTL)
       - `update_status()`: Get task, validate transition via `TaskStatus::can_transition_to()`, update status and timestamp
       - `list()`: Iterate DashMap, filter by owner_id, sort by created_at descending, basic cursor pagination (cursor = last task_id)
       - `cancel()`: Shorthand for update_status to Cancelled
       - `cleanup_expired()`: Iterate DashMap, remove records where `Instant::now() > expires_at`

       Use `dashmap` (already a workspace dependency) for concurrent access. Use `uuid` for task ID generation (already a pmcp-tasks dep, add to pmcp's Cargo.toml with feature `v4`). Use `chrono` for timestamps (already a workspace dependency).
