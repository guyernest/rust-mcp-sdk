# Phase 75.5: PMAT ex-P5 refactor backlog

## Why this phase exists

During Phase 75 Wave 0 (see `.planning/phases/75-fix-pmat-issues/pmat-allow-behavior.md`), we empirically verified that **PMAT 3.15.0 ignores `#[allow(clippy::cognitive_complexity)]`**. The P5 technique documented in `75-CONTEXT.md` (D-02) — "add `#[allow]` + `// Why:` comment and move on" — is dead: allows do not suppress PMAT's complexity gate.

Consequence: functions that Phase 75 planned to resolve via P5 still count as violations. To keep Phase 75 scope-bounded to the work that actually reduces the gate count, we split the P5-dependent work into this sibling phase.

## What's in scope

**Two categories:**

### Category A — Retro-justification sites (migrated wholesale from Plan 75-01 Task 1a-C)

The 13 pre-existing bare `#[allow(clippy::cognitive_complexity)]` sites in `src/` that Plan 75-01 Task 1a-C would have retro-justified. Under D-10-B these allows are ineffective — every site must either (i) be refactored to cognitive complexity ≤25, or (ii) have the pre-existing allow removed because the underlying function already simplified below 25 over time.

### Category B — Escapees from Plans 75-01 through 75-04

Functions that Plans 75-01, 75-02, 75-03, or 75-04 budgeted for P5 (residual cog 26-50 with a Why comment) and that — after exhausting P1/P2/P3/P4 extraction — cannot practically reach ≤25.

Log mechanism: Plans 75-01..75-04 write escapee entries to `.planning/phases/75.5-pmat-ex-p5-refactor-backlog/75.5-ESCAPEES.md` during execution, with:
- File path + function name + line number
- Measured residual cognitive complexity after refactor
- Notes on why ≤25 was impractical
- Commit SHA of the 75-x work that left the escapee

Phase 75.5 planning then uses ESCAPEES.md + the Category A list to scope waves.

## Success criteria

- `pmat analyze complexity --top-files 0 --format json` reports zero violations from Category A sites.
- All Category B escapees are either resolved to ≤25 or re-classified (e.g., test code, generated code, truly irreducible state machine — rare).
- PMAT complexity gate count drops to 0 (in conjunction with Phase 75's work).

## Relationship to Phase 76

Phase 76 (cargo-pmcp IAM declarations) has already landed on `main` without Phase 75's gate enforcement. Phase 75.5 does not block Phase 76 — it's a parallel cleanup track for the ex-P5 backlog.

## Non-goals

- Phase 75.5 does NOT run the CI gate flip. Phase 75 Wave 5 is still the enforcement phase. 75.5 lands before 75-05 to clear the backlog, or in parallel if scope permits.
- Phase 75.5 does NOT touch SATD, duplicates, entropy, or sections. Those remain best-effort within Phase 75 waves.
