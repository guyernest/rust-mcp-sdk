## Issues Encountered

The plan's verify step `cargo test --workspace --all-features` surfaced two pre-existing, environment-dependent test failure clusters that are unrelated to the attribute-removal work:

### Deferred Issues

**1. mcp-e2e-tests::chess (10 failures)** — Pre-existing environmental
- **Cluster:** `chess_initial_status_shows_white_to_move`, `chess_calls_new_game_on_init`, `chess_persists_state_via_bridge`, and 7 other browser-driven tests in `crates/mcp-e2e-tests/tests/chess.rs`.
- **Root cause:** `chromiumoxide v0.9` (with `fetcher` feature) cannot find a Chrome browser archive in the runner cache (`~/.cache/ms-playwright` does not exist). Error: `Installation of browser failed: Failed to open archive: No such file or directory (os error 2)`.
- **Why out of scope:** This plan only deletes `#[allow]` attributes from async/sync functions in `src/server/`, `src/shared/`, `src/client/`. The mcp-e2e-tests crate is a separate workspace member that drives a real headless browser; attribute deletions in pmcp's transport/handler code cannot cause a missing-archive filesystem error.
- **Recommended follow-up:** Run `chromiumoxide`'s fetcher script or install a system Chrome before invoking `cargo test --workspace --all-features` in environments that need the e2e suite. Track separately under a tooling/infra phase if the team wants this gate enabled in CI.

**2. pmcp-tasks::store::redis (24 failures) + pmcp-tasks::store::dynamodb (12 failures) (36 total)** — Pre-existing environmental
- **Cluster:** All `store::redis::integration_tests::*` and `store::dynamodb::integration_tests::*` tests in `crates/pmcp-tasks/`.
- **Root cause:** `Redis connection failed -- is Redis running?: Backend { message: "failed to connect to Redis: Connection refused (os error 61)" }` and analogous DynamoDB connection refused errors. Tests require running Redis on default port + a running LocalStack/DynamoDB endpoint.
- **Why out of scope:** Same as #1 — attribute deletions in pmcp src/ cannot cause TCP "connection refused" against external services. The pmcp-tasks crate is structurally independent of the touched files.
- **Recommended follow-up:** Either (a) gate these tests behind `#[ignore]` or a `--features integration` flag so the bare `cargo test --workspace --all-features` invocation skips them in environments without the services, or (b) ensure CI provisions Redis + DynamoDB Local before invocation. Track separately.

**Verification of out-of-scope classification:** Both clusters fail identically on the parent commit `29dc0a8b` (pre-Phase 75.5) — none of these tests touch any code in `src/server/`, `src/shared/`, or `src/client/`, and the failure messages reference filesystem and TCP errors, not Rust compilation/borrow/lint issues. The CI-matching gate `make quality-gate` (which is the authoritative pre-merge check per CLAUDE.md "Release Steps") exited 0, so phase acceptance is met.
