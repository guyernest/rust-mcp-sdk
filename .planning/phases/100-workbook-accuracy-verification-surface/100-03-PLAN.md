<objective>
Item 2 (WBVER-02): give `render_workbook` an optional `mode: filled | inputs_only` argument (default `filled`, additive). `inputs_only` produces a clean double-entry copy — input cells seeded with the caller's values, formula cells written as BARE formulas with no `set_result` (the server contributes zero output values; Excel computes everything from scratch). An ABSENT mode → Filled; a PRESENT-but-malformed mode value → `Err` (NOT a silent Filled); the encoded `workbook://` URI stays ≤ 64 KiB and round-trips the mode; pre-phase URIs (no mode key) still decode (default Filled).

Threads a new `RenderMode { Filled, InputsOnly }` enum (runtime leaf) from the handler arg, through the `workbook://` URI payload, through `regenerate`, into `render_xlsx(layout, run, mode)`.

Purpose: Double-entry verification download — the BA can pull a formula-only workbook and let Excel be the oracle for arbitrary inputs.
Output: `RenderMode` enum + mode-threaded render + URI payload + handler arg + render-only input schema; per-D-05 a clean copy (no highlighting/formatting/comments).
</objective>
