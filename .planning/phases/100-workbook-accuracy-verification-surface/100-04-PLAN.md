  <behavior>
    - reconcile_reference seeds the executor from the manifest tier defaults, runs it, and for each tool builds a ToolReport whose OutputRows each carry { key, cell: Option<String>, server_value, oracle_value, abs_delta, within_tol }.
    - For a conforming bundle (golden), every row is within_tol and all_within_tol = true at every level.
    - cell = Some(entry.seed_coord.clone()) for every output whose json_key has a matching CellEntry; cell = None only when an oracle key has NO matching outputs entry (malformed bundle) — the row still reports deltas.
    - all_within_tol (report-level) is true iff every checked output across all tools is within TOL; cells_checked counts only rows that were compared (empty-oracle tools contribute 0).
    - A tool with an empty oracle yields ToolReport { outputs: [], all_within_tol: true } (vacuous, D-04).
    - A perturbed oracle (deliberately wrong expected value) yields within_tol = false on that row and all_within_tol = false up the chain.
    - reconcile_reference returns Result (Box<LintFinding> on an executor failure); it NEVER panics/unwraps (deny(panic)).
  </behavior>
