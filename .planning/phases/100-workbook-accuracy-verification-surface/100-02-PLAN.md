<objective>
Item 1 (WBVER-01): make text & boolean formula output cells render as formula-with-cached-result, exactly like numeric formula cells already do. Today the `write_computed_value` Text/Bool arms call `write_string_cell` UNCONDITIONALLY — they ignore `cell.formula`, so Excel's `fullCalcOnLoad="1"` cannot recompute text/bool outputs on open (the trust blind spot this item closes).

Generalize the proven numeric `Formula::new(...).set_result(...)` branch into a shared `write_formula_or_value` helper that covers Number/Text/Bool, and route the Text/Bool arms through it. The helper takes a TYPED per-value-type literal writer (a closure/fn for the non-formula arm) rather than branching on value type inside the helper, keeping it under cog-25 (LOW review item).

Purpose: Close the verification blind spot so ALL output types (not just numeric) carry an independently-recomputable formula+cached-result.
Output: A `write_formula_or_value` helper + text/bool formula cells emitting `<f>`+`<v>`; default behavior for non-formula cells unchanged; NO protocol change.
</objective>
