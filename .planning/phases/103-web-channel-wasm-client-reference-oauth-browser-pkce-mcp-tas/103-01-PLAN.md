<objective>
Add the pure, wasm-safe PKCE crypto helper to the `pmcp` crate (D-02/D-03) and its full
ALWAYS test coverage. This is the strongest property/fuzz target in the phase (pure crypto,
deterministic invariants) and the first reusable public-API piece shipped in the new release.

Purpose: Browser PKCE (and the existing native loopback flow) need RFC 7636 verifier/challenge/
state primitives that compile on wasm32. The native `oauth.rs` primitives are private and use
`rand` (an optional `oauth`-feature dep that won't build on wasm). This helper extracts the logic
with `getrandom::fill` so it is shared and target-agnostic — SDK scope is the crypto helper ONLY
(D-03), no browser orchestrator. Because the module is UNGATED (compiles on host too), the build
correctness hinges on `getrandom` being a NORMAL cross-target dependency, not a wasm-only target
dep (HIGH-1) — today it lives only under `[target.'cfg(target_arch = "wasm32")'.dependencies]`.
Output: `getrandom` relocated to the cross-target `[dependencies]` table, `src/shared/pkce.rs`
(new module), ungated re-export, `tests/pkce_helper.rs` with unit + property + roundtrip coverage,
and a real `cargo fuzz` target (`fuzz/fuzz_targets/pkce_helper.rs`) per CLAUDE.md ALWAYS FUZZ.
</objective>

fn generate_code_verifier() -> String {
    let random_bytes: [u8; 32] = rand::rng().random();   // REPLACE with getrandom::fill

}
fn generate_code_challenge(verifier: &str) -> String {
    let mut hasher = Sha256::new();
    hasher.update(verifier.as_bytes());
    URL_SAFE_NO_PAD.encode(hasher.finalize())
}
