<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser JS ↔ wasm | OAuth orchestration crosses the JS/wasm boundary; tokens live in browser storage |
| browser → IdP/MCP | Authorization redirect, token exchange, and bearer-authenticated MCP requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-103-CSRF | Tampering/Spoofing | OAuth state on redirect return | mitigate | state generated by the pmcp pkce helper, stored in sessionStorage, and verified == returned state in the login-complete method before token exchange |
| T-103-XSS-TOKEN | Information Disclosure | bearer in sessionStorage | accept (documented) | sessionStorage is origin-scoped + cleared on tab close — the demo default (D-06); README documents IndexedDB/httpOnly as the production durability/hardening upgrade |
| T-103-OPENREDIR | Tampering | authorize_url / redirect_uri | mitigate | redirect_uri is the example's own registered URI; the bundled IdP validates it (oauth2.rs:473); the example does not navigate to an attacker-controlled href |
| T-103-RNG | Tampering | verifier/state generation | mitigate | Uses the pmcp pkce helper (getrandom CSPRNG), never Math.random |
| T-103-REPLAY | Spoofing | JSON-RPC id reuse | mitigate | The high-level Client mints unique ids (Uuid::new_v4); the one-shot WasmHttpTransport shares no cache (avoids the MEMORY cross-cache id bug) |
| T-103-SC | Tampering | crate installs | accept | Example deps mirror examples/wasm-client (already vetted); no new external packages; gloo-timers avoided in favor of JS setTimeout |
</threat_model>
