<objective>
GAP 4 (WR-03): In path/query/header position, non-scalar JSON values fall through to
`Value::to_string()` (client.rs `value_to_string` ~293; code_mode.rs `scalar_str` ~759),
emitting JSON with literal `"`/`{`/`[` that then gets percent-encoded into the URL — a
silently-wrong request rather than an error. The `Parameter` schema (http/schema.rs) has
NO `style`/`explode`/`type` hint, so there is no per-param serialization directive to
honor; the defined rule must be uniform.

Purpose: Define and enforce correct behavior for non-scalar params in path/query/header
position. Decision: scalars pass through unchanged; query arrays-OF-SCALARS comma-join
(the existing OpenAPI `form`/`explode:false` behavior, kept); objects, nested
non-scalars, and any non-scalar in path/header position are REJECTED with a clear typed
error naming the param — never silently JSON-stringified. This is the safe, predictable
rule absent a style hint.

Output: A fallible scalar renderer replacing the silent `other => other.to_string()`
fall-through in both client.rs and code_mode.rs, with tests for scalar (unchanged),
array-of-scalars (comma-join), and object/nested-non-scalar (rejected).
</objective>

<decision>
DECIDED RULE (document it in the code): in path/query/header position —
  - scalar (String, Number, Bool, Null) -> bare string (Null -> "null" in code_mode path to preserve existing behavior; in client.rs Null currently routes through `other` — pick the SAME explicit handling: render Null as the empty string OR "null", and document it; match whatever the existing tests assume — check first).
  - query param that is an Array of scalars -> comma-join (KEEP existing build_query behavior).
  - Object, OR an Array containing any non-scalar member, OR a non-scalar in PATH/HEADER position -> REJECT with a typed error naming the param. Do NOT JSON-stringify.
This rule is uniform because http/schema.rs Parameter has no style/explode/type hint to
honor a per-param override. Document the rule in a doc-comment on the renderer.
</decision>
