<task type="auto" tdd="true">
  <name>Task 2: Add OIDC token request and pmcp.run exchange client seam</name>
  <files>cargo-pmcp/src/deployment/targets/pmcp_run/auth.rs</files>
  <read_first>
    - cargo-pmcp/src/deployment/targets/pmcp_run/auth.rs
    - cargo-pmcp/src/deployment/targets/pmcp_run/graphql.rs
  </read_first>
  <action>
    Add `async fn get_credentials_via_github_oidc(env: &GitHubActionsOidcEnv) -> Result<Credentials>`. The function must request a GitHub OIDC JWT from `ACTIONS_ID_TOKEN_REQUEST_URL` with `audience=pmcp.deploy` appended as a query parameter and `Authorization: bearer {ACTIONS_ID_TOKEN_REQUEST_TOKEN}`. Then POST JSON to `{PMCP_API_URL or default}/auth/github-actions/exchange` with fields `oidc_token`, `repository`, `ref`, `workflow`, and `sha`. Parse response fields `access_token`, optional `id_token`, and optional `expires_in` into existing `Credentials`. Use a request timeout of 30 seconds. On non-2xx, return an error containing `GitHub OIDC exchange with pmcp.run failed` and never include the JWT or request token.
  </action>
  <verify>
    <automated>cargo test -p cargo-pmcp github_oidc_exchange</automated>
  </verify>
  <acceptance_criteria>
    - `auth.rs` contains `/auth/github-actions/exchange`
    - Tests cover successful exchange with mock HTTP responses
    - Tests cover rejected exchange and assert error contains `GitHub OIDC exchange with pmcp.run failed`
    - Error strings do not include `oidc_token` contents or `ACTIONS_ID_TOKEN_REQUEST_TOKEN` contents
  </acceptance_criteria>
</task>
