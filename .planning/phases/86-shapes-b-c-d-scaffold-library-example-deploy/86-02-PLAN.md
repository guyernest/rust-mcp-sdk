  `/var/task` is read-only; local `demo.db` otherwise). The example points `PMCP_ASSETS_DIR` at
  its `examples/fixtures/` dir so `cargo run --example` works from any cwd.
- **H2 type-surface fix:** `execute_batch` is called on the CONCRETE `SqliteConnector` BEFORE it is
  wrapped in `Arc<dyn SqlConnector>` for the builder (calling it on `Arc<dyn SqlConnector>` would
  not compile — Gemini Concern §1). The fixture `schema.sql` is IDEMPOTENT
  (`CREATE TABLE IF NOT EXISTS` / `INSERT OR IGNORE`) so a second run against a persisted DB
  succeeds (Codex HIGH).
- **M4 line-count definition:** "≤15 lines" = physical non-empty body STATEMENT lines between
  main's opening `{` and the final `Ok(())`, EXCLUDING imports/attributes/the fn signature/blank
  lines/comment-only lines/the closing brace. The plan and the line-count assertion use the SAME
  definition.

Serves streamable HTTP (D-05, via the Plan 01 `http` feature forward), ships code_mode enabled
with an inline DEV token_secret (D-06). A ChildGuard-protected subprocess spawn-poll integration
test proves initialize → tools/list → tools/call, plus a body-line-count assertion enforces ≤15.

Purpose: prove the line budget, the inline-secret config shape, AND the Lambda-compatible asset/DB
resolver against verified APIs before the scaffold emits them.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client → example HTTP server | untrusted JSON-RPC tools/call params cross into SQL execution |
| config.toml token_secret → code_mode HMAC | inline DEV secret material lives in the fixture |
| PMCP_ASSETS_DIR / pmcp::assets → file reads | the example resolves config/schema through the asset loader (cwd/PMCP_ASSETS_DIR locally) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86-02-01 | Tampering | tools/call `:limit` param → SQL | mitigate | Curated `[[tools]]` use named `:value` params bound via `raw_bind_parameter` (sql/sqlite.rs:160) — never string-concatenated; no `:table` identifier substitution (out of scope). |
| T-86-02-02 | Information Disclosure | inline token_secret in the example fixture | mitigate | The fixture secret is a LOUD DEV-ONLY literal (D-06) requiring `allow_inline_token_secret_for_dev = true`; the comment instructs production to use `env:CODE_MODE_SECRET`. It is an example fixture, not a deploy artifact. |
| T-86-02-03 | Spoofing | DNS rebinding against the local example server | mitigate | Uses `StreamableHttpServer` defaults (`AllowedOrigins::localhost()`, lib.rs:127) — never hand-rolls the HTTP layer (Don't Hand-Roll). |
| T-86-02-04 | Elevation of Privilege | write/DDL on the read-only demo | mitigate | `[code_mode] allow_writes = false` (+ default-deny deletes/ddl) rejects INSERT/DELETE/DROP at validation. |
| T-86-02-05 | Spoofing | forged code-mode approval token | mitigate | HMAC-signed approval tokens (16-byte-min secret via HmacTokenGenerator); execute_code rejects invalid tokens — toolkit machinery, not hand-rolled. |
| T-86-02-06 | Tampering | PMCP_ASSETS_DIR pointing the loader at an attacker path | accept | The example sets PMCP_ASSETS_DIR to its own `CARGO_MANIFEST_DIR/examples/fixtures` (compile-time constant); no external input. The scaffold reads cwd-local assets it itself emitted. Test-/example-only. |
