  scaffold's assets are cwd-local for `cargo run`, and on Lambda the deploy bundle places them under
  `/var/task/assets/` where `pmcp::assets` already looks (Plan 05). So the SAME `main.rs` runs locally
  AND on Lambda with no edits — this is what makes Plan 05's deploy work without touching the entrypoint.

  wrap, and the emitted `schema.sql` is IDEMPOTENT (`CREATE TABLE IF NOT EXISTS` / `INSERT OR IGNORE`)

  any `fs::write` (reject empty, leading digit, characters outside `[A-Za-z0-9_-]`, and any `/`/`..`
