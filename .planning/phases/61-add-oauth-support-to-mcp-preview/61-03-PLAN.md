<objective>
Fix the WASM bridge path so 401/403 from the upstream MCP server reaches the browser as 401/403 (not 502), enabling the OAuthManager re-login flow for the WASM client path.

Purpose: The verification found that forward_raw uses check_response() which converts ALL non-2xx responses (including 401/403) into anyhow::Error. The forward_mcp handler then returns BAD_GATEWAY for all Err variants. This means the browser WASM client receives 502 when the MCP server returns 401/403, and the OAuthManager auth-detection condition (checking response.status === 401 || response.status === 403) never triggers. The non-WASM paths (list_tools, call_tool via send_request) already handle this correctly.

Output: Patched proxy.rs and api.rs with consistent McpRequestError propagation on the forward_raw/forward_mcp path.
</objective>
