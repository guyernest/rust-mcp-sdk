# Phase 60 — UI Design Contract

> Visual and interaction contract for the mcp-preview DevTools panel cleanup. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, gap between tab buttons and Clear All, gap between copy/clear buttons |
| sm | 8px | Network entry margin-bottom, console-time margin-right, inner element spacing |
| xl | 24px | Header icon size (24x24) |

Exceptions:
- **12px (header padding vertical, tab button padding, devtools-content padding):** Not in the standard scale {4, 8, 16, 24, 32, 48, 64}. This is a pre-existing value extracted from the current codebase CSS, not a value introduced by this phase. Changing it would alter the established visual rhythm of the DevTools panel.
- **20px (header padding horizontal):** Not in the standard scale. This is a pre-existing value extracted from the current codebase CSS, not a value introduced by this phase. Changing it would alter the established header layout.
- **Resize handle width: 6px** -- Not a multiple of 4. This is a drag target, not a spacing token. 6px provides adequate grab area without visible intrusion while matching the Chrome DevTools handle width.
- **Touch target minimum: 44px** -- Not in the scale but required for the "Dev Tools" toggle button height per accessibility best practice. The existing button sizes (padding 6px 12px) already achieve this combined with font metrics.

Source: Existing spacing values extracted from `index.html` CSS. This phase does not introduce a new spacing scale; it reuses the established values.

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 13px | 400 (regular) | 1.5 | System stack |
| Label / Tab | 12px | 600 (semibold) | 1.4 | System stack |
| Heading | 18px | 600 (semibold) | 1.2 | System stack |

Font weights used: **2 total** -- 400 (regular) for body text and monospace, 600 (semibold) for headings, tab labels, and interactive controls.

**New elements introduced by this phase:**

| Element | Size | Weight | Line Height | Notes |
|---------|------|--------|-------------|-------|
| "Dev Tools" toggle button text | 13px | 600 (semibold) | 1.0 | Interactive control -- uses semibold to match tab labels |
| "Clear All" button text | 11px | 400 (regular) | 1.0 | Matches existing `.clear-btn` font-size (line 759) |

Source: Extracted from existing CSS -- `.header h1` (18px/600), `.status-indicator` (13px), `.devtools-tab` (12px), `.clear-btn` (11px), `.console-log` (11px/monospace). Tab labels and toggle button consolidated to 600 weight (previously 500 in codebase; promoted to 600 for two-weight constraint).

## Color

All colors use existing CSS custom properties. No new colors are introduced.

| Role | Light Mode | Dark Mode | Usage |
|------|-----------|-----------|-------|
| Dominant (60%) | `--bg-primary` #ffffff | #1a1a1a | Page background, preview area |
| Secondary (30%) | `--bg-secondary` #f5f5f5 | #2d2d2d | Header, tool panel, devtools panel background |
| Accent (10%) | `--accent-color` #0066cc | #3b82f6 | See reserved-for list below |
| Destructive | `--error-color` #ef4444 | #ef4444 | Error entries in network/events tabs |

**Accent reserved for:**
1. Active tab indicator (2px bottom border on selected devtools tab)
2. Active tab text color
3. "Dev Tools" toggle button text when panel is open (`.devtools-toggle.panel-open`)
4. Resize handle hover/drag highlight (at 30% opacity)

**NOT accent:** The "Clear All" button uses `--text-secondary` at rest, `--text-primary` on hover (matching existing clear/copy button pattern). It is not an accent element.

### New color interactions

| Element | State | Color | Source |
|---------|-------|-------|--------|
| Resize handle | Rest | `transparent` | Invisible at rest, matching Chrome DevTools |
| Resize handle | Hover | `var(--accent-color)` at `opacity: 0.3` | Subtle highlight on hover |
| Resize handle | Dragging | `var(--accent-color)` at `opacity: 0.5` | Stronger highlight during active drag |
| "Dev Tools" toggle | Panel open | `var(--accent-color)` | Indicates panel is visible |
| "Dev Tools" toggle | Panel closed | `var(--text-secondary)` | Muted when panel is hidden |
| "Dev Tools" toggle | Hover | `var(--text-primary)` | Standard hover pattern |
| "Clear All" button | Rest | `var(--text-secondary)` | Matches per-tab clear buttons |
| "Clear All" button | Hover | `var(--text-primary)` with `var(--bg-tertiary)` background | Matches per-tab clear buttons |

## Interaction Contract

### 1. Panel Resize (D-02, D-03)

| Property | Value |
|----------|-------|
| Mechanism | Mousedown on resize handle, mousemove adjusts width, mouseup ends |
| Direction | Dragging left increases panel width; dragging right decreases |
| Default width | 350px (D-04) |
| Minimum width | 0px (D-03 -- no minimum, collapses to zero) |
| Maximum width | 80% of viewport width (prevents hiding the entire preview area) |
| Collapse threshold | 0px width triggers `collapsed` class |
| During drag | `user-select: none` on `<body>`, cursor forced to `col-resize` on `<body>` |
| After drag | `user-select` and cursor restored to defaults |
| Transition | None during drag (instant resize). 200ms ease-out transition on toggle open/close. |

### 2. Panel Toggle (D-01)

| Property | Value |
|----------|-------|
| Button location | Header bar, after `.theme-toggle`, before closing `</header>` |
| Click when open | Sets panel width to 0px, adds `.collapsed` class, transitions in 200ms |
| Click when closed | Sets panel width to 350px (default), removes `.collapsed`, transitions in 200ms |
| State indicator | Button text color: `--accent-color` when open, `--text-secondary` when closed |
| Panel memory | None (D-05). Always resets to 350px open on page reload. |

### 3. Global Clear All (D-07, D-08, D-09)

| Property | Value |
|----------|-------|
| Button location | Inside `.devtools-tabs` bar, right-aligned (pushed with `margin-left: auto`) |
| Behavior | Programmatically clicks each per-tab clear button for all 4 tabs |
| Tab clear order | network, events, protocol, bridge (order does not matter, all synchronous) |
| Button style | Matches `.clear-btn` styling (11px, border, transparent bg, secondary text) |
| No confirmation | Executes immediately; debug logs are ephemeral |

### 4. Tab Default (after Console removal)

| Property | Value |
|----------|-------|
| Default active tab | Network (first in remaining list: Network, Events, Protocol, Bridge) |
| Tab switching | Unchanged -- `data-tab` attribute click handler, `classList.toggle('active')` |
