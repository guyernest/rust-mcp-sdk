<objective>
Add theme CSS variable palettes to mcp-preview so widgets using the ext-apps SDK theme system can respond to light/dark theme toggling.

Purpose: mcp-preview currently sends `theme: "light"|"dark"` in host context but omits `styles.variables` — the CSS custom properties from the ext-apps `McpUiStyles` type. Widgets that use `applyHostStyleVariables(ctx.styles.variables)` receive undefined, making theme toggling visually invisible. This makes theme testing during widget development impossible.

Output: Updated index.html with THEME_PALETTES constant and styles.variables included in host context at init and on theme toggle.
</objective>

Reference for CSS variable names (ext-apps SDK types):
- McpUiStyleVariableKey defines all valid CSS variable names
- applyHostStyleVariables() sets each key as a CSS custom property on document.documentElement
- ChatGPT sends ~80 variables; we need the core subset that widgets actually reference

Reference implementation pattern (open-images widget):
```javascript
// Widget CSS pattern:
// background: var(--color-background-primary, #1a1a1a);
// The fallback (#1a1a1a) is used ONLY when the host doesn't provide the variable.
// With THEME_PALETTES, mcp-preview provides the variables, making theme toggling work.
```

**Design notes:**
- Colors that differ: backgrounds, text, borders, rings, shadows
- Values that are the same: fonts, text sizes, heading sizes, border-radius — these are theme-agnostic
- Dark shadows use stronger opacity since dark backgrounds need more shadow contrast
- Color values are chosen to match the ext-apps SDK default CSS fallbacks in widgets (e.g., open-images uses `var(--color-background-primary, #1a1a1a)` for dark)
