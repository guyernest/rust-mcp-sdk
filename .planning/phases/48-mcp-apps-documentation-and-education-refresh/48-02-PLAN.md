Add section "## App Metadata Validation" with:

1. **Learning objective:** "After this section, you will be able to validate MCP App metadata compliance using mcp-tester apps and cargo pmcp test apps"

2. **Why it matters:** MCP Apps servers must emit correct _meta, MIME types, and resource cross-references for widgets to render across different hosts. Manual inspection is error-prone.

3. **CLI usage examples:**
   ```bash
   # Standard validation
   mcp-tester apps http://localhost:3000

   # ChatGPT-specific validation
   mcp-tester apps http://localhost:3000 --mode chatgpt

   # Claude Desktop validation
   mcp-tester apps http://localhost:3000 --mode claude-desktop

   # Strict mode (warnings become failures)
   mcp-tester apps http://localhost:3000 --strict

   # Via cargo pmcp
   cargo pmcp test apps --url http://localhost:3000 --mode chatgpt --strict
   ```

4. **What gets validated** (brief list):
   - _meta field present on App-capable tools
   - ui.resourceUri points to valid widget resource
   - MIME type is `text/html;profile=mcp-app` (not legacy variants)
   - Resource cross-reference (tool URI matches resources/list)
   - [ChatGPT mode] openai/* descriptor keys present
   - [if present] outputSchema structure valid

5. **Try it** callout: "Run `mcp-tester apps` against the Open Images example server and examine the validation output"

6. **Knowledge check:** "What validation mode would you use to ensure your MCP App works with Claude Desktop?"

Do NOT change existing sections of ch11-02-mcp-tester.md -- only add the new App metadata content.
