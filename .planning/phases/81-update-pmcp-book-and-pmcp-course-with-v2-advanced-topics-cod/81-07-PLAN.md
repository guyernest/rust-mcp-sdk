    **Audit A — Inline excerpt drift (STRENGTHENED per revision R-1):**

    Codex HIGH-1 and Gemini HIGH-1 both flagged the previous heuristic — single-substring grep with "nearest preceding Full example link" heuristic — as fragile. Audit A is rebuilt around two changes:

    1. **Explicit anchor (Gemini's suggestion):** The file-to-excerpt mapping is established by the markdown pattern `Full example: [`<path>`](https://github.com/paiml/rust-mcp-sdk/blob/main/<path>)`. Excerpts that are not preceded (within the same H2 section) by this pattern are NOT subject to Audit A (they are either synthetic or out-of-scope; see opt-out below). This increases precision when an H2 section contains multiple excerpts from different sources — each "Full example:" anchor scopes the immediately-following block(s) to that source.

    2. **Contiguous-block matching (Codex's suggestion):** Instead of single-substring grep, extract the entire fenced block contents, normalize trailing whitespace (and trailing-whitespace-only lines), and assert that the entire normalized block appears as a contiguous substring of the cited source file.

    3. **Synthetic-snippet opt-out:** A fenced block immediately preceded (no intervening content other than blank lines) by an HTML comment `<!-- synthetic -->` is COUNTED AS "skipped synthetic" but NOT a failure. This handles `config.toml` templates and prose-only examples that aren't excerpts at all. Synthetic snippets are tracked separately in the report.

    Algorithm (pseudo-shell — the executor may implement as a small Python/awk script):
    ```
    for each chapter in $CHAPTERS:
      parse the chapter into a sequence of H2 sections.
      for each H2 section:
        scan for `Full example: [` followed by a backtick-wrapped path and a URL.
        if found, record CURRENT_SOURCE = the path inside the backticks.
        for each fenced block in the section:
          look at the LINE IMMEDIATELY PRECEDING the opening fence (skip blank lines only).
          if it equals `<!-- synthetic -->`:
            increment SYNTHETIC_COUNT; do not check.
          else if CURRENT_SOURCE is set:
            extract block contents (everything between opening and closing fence).
            normalize: strip trailing whitespace from each line; collapse runs of trailing-empty lines.
            read CURRENT_SOURCE; normalize the same way.
            assert block contents appears as a contiguous substring of source.
            if not:
              record FAIL: "<chapter>:<line> excerpt drift — block does not match <CURRENT_SOURCE> contiguously"
              quote the first 5 lines of the block + the first non-matching line.
            else:
              increment EXCERPT_OK_COUNT.
          else:
            increment EXCERPT_UNANCHORED_COUNT (no Full example: link in scope; not subject to Audit A).
    ```

Implementation (markdown tables conventionally start with `|` on each row; the header row is followed by an `|---|---|` separator row, and content rows follow):

    extract_language_table_rows() {
      local file="$1"
      # Find a markdown table whose header row's first column is "Language" (case-insensitive).
      # Return the count of content rows (excluding the header and separator).
      awk '
        BEGIN { in_table=0; rows=0; }
        /^\| *[Ll]anguage *\|/ {
          in_table=1; next;
        }
        in_table &amp;&amp; /^\|[-: |]+\|$/ {
          # separator row; skip
          next;
        }
        in_table &amp;&amp; /^\|/ {
          rows++;
        }
        in_table &amp;&amp; !/^\|/ {
          in_table=0;
        }
        END { print rows }
      ' "$file"
