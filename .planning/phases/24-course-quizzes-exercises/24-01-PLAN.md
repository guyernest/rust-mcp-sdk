**Questions to include (~10 total, mix of MultipleChoice and ShortAnswer):**

The questions MUST cover these topics from ch18-03-performance.md:

1. **`cargo pmcp loadtest` purpose** — Why use MCP-specific load testing vs generic HTTP tools like wrk or ab? (Answer: understands MCP protocol semantics — initialize handshakes, typed tools/call requests, per-operation HdrHistogram metrics)
2. **Schema discovery** — What does `cargo pmcp loadtest init <url>` do? (ShortAnswer, answer: connects to running server, discovers tools/resources/prompts, generates TOML config automatically)
3. **TOML config structure** — What are the three scenario types in loadtest config? (Answer: tools_call, resources_read, prompts_get)
4. **Flat vs staged execution** — What is the difference between flat and staged load profiles? (Answer: flat holds constant VUs for a duration; staged ramps VUs through multiple stages for progressive load increase)
5. **HdrHistogram percentiles** — Why is P99 latency more useful than average for load testing? (Answer: average hides tail latency; P99 shows worst-case experience for 1% of requests)
6. **Coordinated omission** — What is coordinated omission and why does `cargo pmcp loadtest` correct for it? (Answer: when slow responses cause the test to send fewer requests, hiding true latency; correction records what latency would have been if requests were sent on schedule)
7. **Breaking point detection** — How does the loadtest tool detect a server's breaking point? (Answer: error rate exceeds threshold OR P99 latency exceeds configured limit during staged ramp-up)
8. **Virtual users (VUs)** — What does the `virtual_users` parameter control? (Answer: number of concurrent simulated clients each maintaining their own connection and sending requests independently)
9. **`loadtest init` convenience** — What flag generates a ready-to-run config without manual editing? (ShortAnswer-style or MC about the init workflow)
10. **Capacity planning** — What is the recommended approach to determine production capacity from load test results? (Answer: find the breaking point, apply a safety margin like 70% of breaking point as target capacity)

**Format rules (MUST follow exactly):**

- MultipleChoice: `prompt.distractors` (array of 3), `answer.answer`, `answer.position` (0-3)

- Distractors must be plausible but clearly wrong, similar length to correct answer
