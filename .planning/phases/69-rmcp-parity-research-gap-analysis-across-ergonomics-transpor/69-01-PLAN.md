The gap-matrix table shape is locked. Every H2 surface section uses exactly this column set, in this order:

| Surface subtopic | rmcp approach | rmcp evidence | pmcp approach | pmcp evidence | Gap | Severity |
|------------------|---------------|---------------|---------------|---------------|-----|----------|

- Surface subtopic: a specific ergonomic aspect of the surface (e.g., for Tool Macros: "async fn support", "attribute-parameter richness", "schema derivation", "error-type conversion", "state injection").
- rmcp approach: 1–3 sentence description. Factual, not judgmental. Phrased as "what rmcp does", not "rmcp is better/worse at".
- rmcp evidence: file path + line number from the rmcp source repo at the pinned tag, OR a rustdoc anchor URL, OR a GitHub blob URL with `#L<n>` fragment. Bare URLs without line numbers are NOT acceptable.
- pmcp approach: 1–3 sentence description of the pmcp equivalent.
- pmcp evidence: file path + line number (e.g., `src/server/builder.rs:142`). Line numbers MUST match the repo state at the HEAD commit used for this research. Record the commit SHA in the document header.
- Gap: one of three values: (a) a specific gap description (e.g., "pmcp lacks attribute-level `#[tool(description = ...)]` override; description is derived from rustdoc only"), (b) "Parity" if both SDKs take the same approach, (c) "Strength to preserve" if pmcp does it better than rmcp.
- Severity: exactly one of "High", "Medium", "Low", per the D-14 rubric:
  - **High**: affects the typical first-hour user path AND a clean specific fix can be articulated.
  - **Medium**: affects less common paths or has no single clean fix.
  - **Low**: cosmetic or theoretical. "Parity" rows and "Strength to preserve" rows are always Low.
