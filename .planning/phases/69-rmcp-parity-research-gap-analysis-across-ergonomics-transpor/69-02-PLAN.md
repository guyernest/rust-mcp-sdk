**REQ-ID naming convention:** Exactly ONE ID per proposal. Use the prefix `PARITY-` followed by a surface code and the two-digit counter `01`:
- Tool macros proposals: `PARITY-MACRO-01`
- Builder proposals: `PARITY-BUILDER-01`
- Typed wrapper proposals: `PARITY-TYPED-01`
- Handler/state proposals: `PARITY-HANDLER-01`
- Client ergonomics proposals: `PARITY-CLIENT-01`
- Error-type proposals: `PARITY-ERR-01`

The surface prefix is determined by the proposal's **dominant surface** — the surface containing the majority of Row IDs the proposal addresses. If a proposal addresses multiple High rows from different surfaces (rare; usually only when genuinely cross-cutting), pick the surface with the most cited rows; ties broken by the surface-order precedence MACRO > BUILDER > TYPED > HANDLER > CLIENT > ERR.

If two proposals would otherwise share the same prefix (e.g., two separate Tool Macros proposals), increment the counter — the second becomes `PARITY-MACRO-02`, a third becomes `PARITY-MACRO-03`. IDs are unique across the whole document. Plan 03 cross-checks that these IDs land in REQUIREMENTS.md.
