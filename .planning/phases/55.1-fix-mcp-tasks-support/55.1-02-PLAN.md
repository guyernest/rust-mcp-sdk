<objective>
Wire task detection into ServerCore's `handle_call_tool` so that tools on the standard `task_store` path (not the legacy `TaskRouter` path) return proper `CreateTaskResult` wire format instead of wrapping Task JSON as `CallToolResult` text.

Purpose: Gap 2 and Gap 3 from investigation -- the SDK currently wraps all tool handler returns as `CallToolResult`, even when the tool creates a task via `task_store`. Clients see serialized Task JSON as text content and have no protocol-level signal to begin `tasks/get` polling. This change makes the standard task_store path produce the correct wire format.

Output: core.rs detects task-creating tools and returns `CreateTaskResult` with `_meta` containing `io.modelcontextprotocol/related-task`.
</objective>
