2. Create `crates/mcp-tester/src/app_validator.rs` with:

```rust
pub enum AppValidationMode {
    Standard,       // ui.resourceUri nested key only
    ChatGpt,        // + openai/outputTemplate, openai/toolInvocation/*, legacy flat ui/resourceUri
    ClaudeDesktop,  // same as Standard for now (per research recommendation)
}

pub struct AppValidator {
    mode: AppValidationMode,
    strict: bool,
    tool_filter: Option<String>,
}
```

Implement the following methods on `AppValidator`:

- `new(mode, strict, tool_filter)` constructor
- `validate_tools(&self, tools: &[ToolInfo], resources: &[ResourceInfo]) -> Vec<TestResult>` -- main entry point:
  - Filter to App-capable tools (those with `_meta` containing nested `ui.resourceUri` OR flat `ui/resourceUri` key)
  - If `tool_filter` is Some, only validate the named tool
  - For each App-capable tool, call `validate_tool_meta()` and `validate_resource_match()`
  - If mode is ChatGpt, also call `validate_chatgpt_keys()`
  - If `output_schema` is present on the tool, call `validate_output_schema()`
  - Return all TestResult entries with `TestCategory::Apps`

- `is_app_capable(tool: &ToolInfo) -> bool` (pub static/associated function):
  - Check `_meta.ui.resourceUri` (nested) OR `_meta["ui/resourceUri"]` (flat legacy)
  - Return true if either exists and is a string

- `validate_tool_meta(&self, tool: &ToolInfo) -> Vec<TestResult>`:
  - Check `_meta.ui.resourceUri` exists (Passed/Failed)
  - Validate URI format using `url::Url::parse()` -- note `ui://app/` is a custom scheme, so just check it's a non-empty string with a scheme separator (Warning if not parseable, not failure)

- `validate_resource_match(&self, tool_name: &str, resource_uri: &str, resources: &[ResourceInfo]) -> Vec<TestResult>`:
  - Find resource with matching URI
  - If not found: Warning (NOT failure per user decision) with details message
  - If found: validate MIME type is one of `text/html`, `application/html+mcp-app`, `text/html;profile=mcp-app` (case-insensitive)
  - MIME type mismatch is Warning with details showing actual vs expected

- `validate_chatgpt_keys(&self, tool_name: &str, meta: &Map) -> Vec<TestResult>`:
  - Check for: `openai/outputTemplate`, `openai/toolInvocation/invoking`, `openai/toolInvocation/invoked`, `openai/widgetAccessible`
  - Also check flat legacy key `ui/resourceUri` exists
  - Missing keys are Warnings (not failures)

- `validate_output_schema(&self, tool_name: &str, schema: &Value) -> Vec<TestResult>`:
  - Check it's an object with a `type` field (basic structural check per research)
  - Pass if valid structure, Warning if malformed

All TestResult entries use `TestCategory::Apps`. Duration is `Duration::from_secs(0)` for metadata-only checks.
