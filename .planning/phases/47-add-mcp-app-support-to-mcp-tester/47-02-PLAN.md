Implementation:
- Parse mode string: None or "standard" -> Standard, "chatgpt" -> ChatGpt, "claude-desktop" -> ClaudeDesktop, other -> anyhow::bail with error listing valid modes

- Call `validator.validate_tools(&tools_result.tools, &resources_result.resources)`
- If result is empty (no App-capable tools): print info message "No App-capable tools found on this server" and return Ok(())
- Build `TestReport` from results via `report.add_test()` for each TestResult
- If strict: call `report.apply_strict_mode()`
- Print report using existing format (Pretty output with per-tool grouping)
- If `report.has_failures()`: bail with error message
- Otherwise: print success summary and return Ok(())

```rust
/// Validate MCP App metadata compliance
///
/// Checks tools for App-capable metadata (ui.resourceUri), validates MIME types,

Apps {
    /// URL of the MCP server to validate
    #[arg(long, required = true)]
    url: String,

    /// Validation mode: standard, chatgpt, or claude-desktop
    #[arg(long)]
    mode: Option<String>,

    /// Test specific tool only
    #[arg(long)]
    tool: Option<String>,

    /// Strict mode (promote warnings to failures)
    #[arg(long)]
    strict: bool,

    /// Transport type: http, jsonrpc, or stdio
    #[arg(long)]
    transport: Option<String>,

    /// Show verbose output
    #[arg(long, short)]
    verbose: bool,

    /// Connection timeout in seconds
    #[arg(long, default_value = "30")]
    timeout: u64,
},
```
